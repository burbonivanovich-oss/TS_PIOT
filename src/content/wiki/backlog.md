### Задача 33 — направления для проработки

Контент-план сейчас построен на классических информационных статьях: «что такое», «как подключить», «штрафы за». Это даёт трафик, но легко копируется и не отстраивает портал. Нужно подумать, какие форматы добавить — в идеале такие, которые одновременно дают SEO-вес, удерживают на странице и закрывают конкретный сценарий пользователя.

**Возможные направления (нужно обсудить и приоритизировать):**

1. **Интерактивные инструменты под кластер** — по образцу уже существующих `/kalkulyator-shtrafov/` и `/kalendar-markirovki/`. Кандидаты:
   - Подбор кассы по сценарию бизнеса (кластер ККТ): ниша → оборот → маркировка → 2–3 модели.
   - Калькулятор УСН/НДС с 2026 (выбор ставки 5/7/20%).
   - Симулятор перехода ИП → ООО с расчётом налоговой разницы.
   - Чек-лист готовности к ТС ПИоТ с проверкой по шагам.
   - Подбор провайдера ТС ПИоТ / ОФД по параметрам.

2. **Сценарные гайды** вместо «что такое X» — путь под конкретную ситуацию: «Открываю кофейню — что мне нужно за 30 дней», «Перехожу с ЕНВД на УСН в 2026», «Получил требование ФНС — пошаговый ответ». Линейный таймлайн с чек-боксами.

3. **Сравнительные таблицы с фильтрами** — React-компонент (как `BlogFilter.tsx`): сравнение моделей касс, ФН, ОФД, банков для ИП по параметрам с реальной фильтрацией.

4. **Разборы реальных кейсов** — анонимизированные истории МСБ с цифрами: штрафы, споры с ФНС, переход на маркировку. Source: судебная практика sudact.ru, разъяснения, кейсы из Telegram-каналов. Формат — «ситуация → ошибка → последствия → как было правильно».

5. **Визуальные explainer-материалы** — схемы дата-флоу (как чек идёт от кассы → ОФД → ФНС → ТС ПИоТ → ЧЗ), таймлайны нормативных изменений, инфографика по штрафам. Картинки от OpenRouter/FLUX уже есть в инфраструктуре.

6. **Q&A-блоки под AI Overviews** — структурированные FAQ-секции с короткими ответами (40–60 слов) под цитирование в ChatGPT/Perplexity/Google AI. Уже частично умеет `/blog geo`.

7. **Обновляемые «живые» страницы** — одна страница, которая постоянно актуализируется: «Календарь маркировки 2026» (уже есть), «Текущие лимиты УСН», «Актуальные штрафы за маркировку». Дата `updatedDate` + явная плашка «обновлено DD.MM».

8. **Шаблоны документов** — заявления, уведомления в ФНС, чек-листы для скачивания. Лид-магниты с осторожностью: только если есть правовая проверка.

9. **Видео/аудио** — короткие YouTube-разборы по 3–5 мин на статью, подкаст-эпизоды. `/blog audio` через Gemini TTS уже подключён, но требует Google API key.

10. **Партнёрские разборы с ФНС/ЧЗ** — официальные комментарии ведомств под публикацию. Долгий, но сильно поднимает E-E-A-T.

**Вопросы, на которые нужно ответить перед началом работы:**
- Какие 2–3 формата дают максимальный ROI: трафик + удержание + отстройка?
- Какие кластеры приоритетны для эксперимента (ТС ПИоТ, маркировка 2026, старт бизнеса)?
- Нужна ли отдельная категория `/instrumenty/` или интерактивы встраиваются в статьи?
- Какие компоненты переиспользуемы (фильтры, калькуляторы, таймлайны)?

**Следующий шаг:** выбрать 1–2 формата для пилота, прогнать через один кластер, замерить метрики (время на странице, прокрутка, переходы по внутренним ссылкам).
