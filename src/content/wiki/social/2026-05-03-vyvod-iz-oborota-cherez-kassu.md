## Telegram

Каждая продажа маркированного товара через кассу — это вывод кода DataMatrix из системы «Честный знак». Если этого не происходит, товар числится у вас в обороте и после продажи — это нарушение.

Цепочка: кассир сканирует DataMatrix → ТС ПИоТ проверяет статус кода в «Честном знаке» → при успехе продажа разрешена, код попадает в чек ФФД 1.2 → данные через ОФД уходят в ФНС → «Честный знак» автоматически выводит код из оборота.

Для работы нужны: касса с ФФД 1.2, активированный ТС ПИоТ, 2D-сканер. Без хотя бы одного из трёх — вывод из оборота не происходит корректно.

## VK

Продаёте маркированные товары? Важно понимать, что «вывод из оборота» — это не абстрактная операция, а конкретный технический процесс, который должен происходить при каждой продаже через кассу.

Вот как это работает по шагам: кассир сканирует код DataMatrix с упаковки → кассовое ПО передаёт код в модуль ТС ПИоТ → ТС ПИоТ обращается к «Честному знаку» с запросом «можно ли продать» → при ответе «разрешено» продажа завершается → код записывается в фискальный чек → данные через ОФД поступают в ФНС и «Честный знак» → код автоматически аннулируется.

Если этой цепочки нет — вы продаёте товар, который формально продолжает числиться в обороте. При проверке это выявляется сопоставлением данных ОФД и «Честного знака».

Три условия для корректной работы: касса поддерживает ФФД 1.2, ТС ПИоТ установлен и активирован, 2D-сканер правильно передаёт код в ПО.

## Дзен

**Заголовок Дзен:** Вывод из оборота через кассу: три ошибки, которые нарушают прослеживаемость маркировки

Магазин сканирует коды DataMatrix при продаже, данные уходят в ОФД — казалось бы, всё в порядке. Но «Честный знак» при этом может не видеть вывода из оборота. Разберём, где чаще всего возникают разрывы в цепочке.

Ошибка первая: ТС ПИоТ не установлен или отключён. Касса формирует чек с кодом — но модуль, который должен передать данные в «Честный знак», не работает. Код записывается в чек ФФД 1.2, но вывода из оборота в системе прослеживаемости не происходит.

Ошибка вторая: сканер передаёт код с лишними символами. Это технический момент: некоторые сканеры добавляют префиксы или символы-разделители. ТС ПИоТ получает «сломанный» код и не может идентифицировать товар — продажа проходит, вывода нет.

Ошибка третья: возврат без восстановления кода в обороте. Покупатель вернул товар. Если не оформить чек возврата с кодом DataMatrix — товар остаётся выведенным из оборота, хотя физически снова у вас на полке. Повторно продать его уже нельзя.

Как проверить, что цепочка работает корректно, и как исправить каждую из ошибок — в полной статье.
