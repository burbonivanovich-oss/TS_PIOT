## Telegram

Каждая продажа маркированного товара через кассу — это вывод кода DataMatrix из системы «Честный знак». Если этого не происходит, товар числится у магазина в обороте даже после продажи — это нарушение, которое фиксируется при сопоставлении данных ОФД и системы прослеживаемости.

Цепочка: кассир сканирует DataMatrix → ТС ПИоТ проверяет статус кода в «Честном знаке» → при ответе «разрешено» продажа проходит, код записывается в чек ФФД 1.2 → данные через ОФД уходят в ФНС → «Честный знак» автоматически аннулирует код.

С 28 декабря 2025 года ТС ПИоТ обязателен для всех точек с маркированным товаром (ст. 4.3 54-ФЗ). Для корректной работы нужны три элемента: касса с ФФД 1.2, активированный ТС ПИоТ, 2D-сканер DataMatrix.

## VK

Вывод из оборота — это операция, которой маркированный товар выходит из системы «Честный знак» при продаже покупателю. Если она не происходит, товар продолжает числиться в остатках магазина в системе прослеживаемости. При проверке это выявляется сопоставлением данных ОФД и «Честного знака».

Как работает вывод через кассу

1. Кассир сканирует код DataMatrix с упаковки
2. Кассовое ПО передаёт код в ТС ПИоТ
3. ТС ПИоТ запрашивает «Честный знак»: разрешён ли товар к продаже
4. При ответе «разрешено» — продажа проходит, код записывается в чек по ФФД 1.2
5. Данные через ОФД поступают в ФНС и «Честный знак» — код автоматически аннулируется

С 28 декабря 2025 года шаги 2–3 обязательны для всех точек с маркированным товаром (ст. 4.3 54-ФЗ).

Как настроить кассу

Шаг 1: убедиться, что касса поддерживает ФФД 1.2 — без этого код маркировки не попадёт в чек.
Шаг 2: активировать ТС ПИоТ. На Эвоторе — через магазин приложений. На других платформах — через кассовое ПО.
Шаг 3: проверить настройку 2D-сканера — он должен передавать код DataMatrix без лишних символов.
Шаг 4: провести тестовую продажу с реальным кодом и убедиться, что в чеке есть раздел «Предмет расчёта» с кодом маркировки.

Типичные ошибки:
— ТС ПИоТ не установлен: данные уходят в ОФД, но вывода из оборота нет
— Возврат оформлен без восстановления кода в обороте
— Товар утилизирован без вывода через личный кабинет «Честного знака»

## Дзен

**Заголовок Дзен:** Вывод из оборота через кассу: почему маркированный товар не уходит из системы

Магазин сканирует коды DataMatrix при продаже, данные уходят в ОФД — казалось бы, всё в порядке. Но «Честный знак» при этом может не видеть вывода из оборота. Разберём, как устроен механизм и где чаще всего возникают разрывы.

Типичный сценарий «оно работало». Магазин косметики в Подмосковье отчитал три месяца продаж, на полке остаток в 40 позиций. Открывают личный кабинет «Честного знака» — а там в обороте числится 380 единиц. То есть 340 уже проданных пачек система не вывела. Причина — кассир сканировал DataMatrix, а ТС ПИоТ не был активирован после обновления прошивки. Касса печатала чеки с кодом, но запрос в «Честный знак» не уходил. Восстановление заняло три недели: пришлось вручную выводить 340 кодов с обоснованием «продано» и объяснительной запиской в ЦРПТ.

Что такое вывод из оборота

Каждый маркированный товар с момента производства или ввоза числится в «Честном знаке» как «в обороте». Он передаётся от производителя к дистрибьютору, от дистрибьютора к магазину — каждый переход фиксируется через ЭДО. Когда товар продаётся конечному покупателю — код должен аннулироваться. Если этого не происходит, инспекция «Честного знака» выявит расхождение при сопоставлении данных.

Механизм на кассе

При продаже кассир сканирует код DataMatrix. Кассовое ПО передаёт его в ТС ПИоТ — обязательный с 28 декабря 2025 года программный модуль (ст. 4.3 54-ФЗ). ТС ПИоТ направляет запрос в «Честный знак» и получает ответ: разрешено или нет. При положительном ответе код записывается в фискальный чек по ФФД 1.2, данные через ОФД поступают в ФНС, а «Честный знак» автоматически аннулирует код.

При разрешительном режиме (введён для ряда категорий) касса физически не может завершить продажу без разрешения системы.

Настройка кассы: четыре шага

Первый: убедиться, что касса поддерживает ФФД 1.2. Без этого формата код маркировки не попадёт в чек, и вывод из оборота не произойдёт.

Второй: активировать ТС ПИоТ. На Эвоторе — через магазин приложений. На других платформах — через кассовое ПО или отдельную установку.

Третий: проверить настройку 2D-сканера. Некоторые модели добавляют к коду лишние символы или префиксы — ТС ПИоТ не сможет идентифицировать такой код.

Четвёртый: провести тестовую продажу с реальным кодом DataMatrix и убедиться, что в чеке присутствует раздел «Предмет расчёта» с кодом маркировки.

Ошибки, которые встречаются чаще всего

Самая распространённая: ТС ПИоТ не установлен или деактивирован. Касса работает, данные в ОФД уходят, код в чеке есть — но «Честный знак» не получает сигнала о выводе из оборота.

Вторая: возврат оформлен без восстановления кода. При возврате товара его код должен вернуться в оборот — это делается через кассовое ПО при оформлении чека возврата с кодом DataMatrix.

Третья: товар утилизирован или испорчен без вывода из оборота. Код продолжает числиться в остатках — нужно вывести его в личном кабинете «Честного знака» с указанием основания «утилизация».

Как проверить, что вывод действительно работает. Раз в неделю заходите в личный кабинет «Честного знака», раздел «Документы → Вывод из оборота». Сопоставьте количество выведенных кодов с количеством чеков с маркировкой в ОФД. Расхождение больше 2–3% — сигнал, что часть продаж проходит мимо ТС ПИоТ. Если расхождение нашли — не ждите проверки: оформите ретроспективный вывод и приложите пояснение, это снимает риски по ст. 15.12 КоАП РФ.
