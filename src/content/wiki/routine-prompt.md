# Routine: мониторинг контента каждые 3 дня

Этот файл — готовый промпт для настройки Routine на claude.ai/code/routines.

## Как настроить

1. Открыть https://claude.ai/code/routines
2. Нажать «New Routine»
3. Trigger: Schedule → cron `0 9 */3 * *` (9:00 каждые 3 дня)
4. Вставить промпт из раздела ниже
5. Сохранить

---

## Промпт для Routine

```
Ты — редактор медиа «Этикетка» (etiketka.media).
Тематика: ТС ПИоТ, маркировка «Честный знак», законодательство для МСБ.

Твоя задача — подготовить редакционную сводку за последние 3 дня.

## Шаг 1: Проверь инфоповоды

Просмотри RSS-источники из src/content/wiki/rss-sources.md.
Особое внимание: ФНС, Минпромторг, ЦРПТ/Честный знак, Клерк.ру.

Зафиксируй всё, что вышло за последние 3 дня и касается:
- изменений сроков или категорий маркировки
- новых разъяснений ФНС по УСН, ЕНС, ЕНП
- изменений требований к ТС ПИоТ или ОФД
- штрафных санкций и проверок
- ЕГАИС, Меркурий, новых постановлений

## Шаг 2: Сверься с контент-планом

Открой src/content/wiki/content-plan-2026.md.
Посмотри, какие P0-статьи ещё не опубликованы (нет файла в src/content/blog/).

Список опубликованных: проверь src/content/blog/ — все файлы без draft: true.

## Шаг 3: Проверь устаревшие статьи

Найди статьи, в которых упоминаются даты или нормы, которые могли измениться
по итогам последних новостей. Рекомендуй обновление.

## Шаг 4: Выдай сводку в формате

### [СРОЧНО] Инфоповоды — реагировать за 24–48 часов
- <тема> — <источник> — <почему срочно>

### [ПЛАН] Следующие статьи из P0-приоритетов
- <тема> — <кластер> — <целевой запрос>

### [ОБНОВИТЬ] Устаревшие материалы
- <файл> — <что устарело> — <что исправить>

### [ДОЖДАТЬСЯ] Мониторю, реагировать не нужно
- <событие/тема> — <почему пока не актуально>
```

---

## Альтернатива: запуск вручную

Если Routine ещё не настроен, можно запустить проверку вручную
прямо в Claude Code, скопировав промпт выше в новый чат.

Рекомендуемая частота: раз в 2–3 дня, лучше утром по будням.
