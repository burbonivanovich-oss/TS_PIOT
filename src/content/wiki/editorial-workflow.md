# Редакционный workflow: форматы и процессы

## Форматы контента

### 1. Полноформатная статья (1500–2500 слов)

Опорные статьи кластеров. Запуск:

```
/new-post "тема"
```

Пайплайн: research-specialist → content-writer → seo-optimizer.
После: `/blog factcheck <файл>`, `/blog seo-check <файл>`, ставим `draft: false`.

---

### 2. Быстрый ответ (400–600 слов)

Короткие страницы под конкретные НЧ-запросы. Целевой формат для featured snippets.

**Когда использовать:** запрос типа «штраф за X 2026», «нужен ли X для Y», «как сделать X пошагово».

**Структура:**
```
H1: [Конкретный вопрос или факт с цифрой]
Первый абзац: прямой ответ (1–2 предложения с ключевой цифрой/фактом)
H2: Кого касается
H2: [Цифры/таблица/список шагов]
H2: Что делать
[3 внутренние ссылки на полноформатные статьи]
```

**Frontmatter:** добавить тег `быстрый-ответ`, длина description 140–160 символов с цифрой.

**Запуск:** писать напрямую (без полного пайплайна), проверить `/blog seo-check`.

---

### 3. Практический кейс (800–1200 слов)

Разбор ситуации конкретного типа бизнеса. Низкая конкуренция, высокий спрос.

**Персоны для кейсов:**
- ИП в кафе/общепите (алкоголь + ЕГАИС + маркировка)
- Продавец на Wildberries/Ozon (маркировка + документы)
- Розничный магазин одежды/обуви (ТС ПИоТ + маркировка)
- ИП на патенте (нужен ли ТС ПИоТ, что изменилось)
- Небольшой импортёр (ввод в оборот + Честный знак)
- Микропредприятие на УСН (налоги + ЭДО + ЕНС)

**Структура:**
```
H1: [Персона]: что изменилось и что делать в 2026
Лид: профиль персоны, 2–3 предложения
H2: Что требует закон (конкретно для этого типа бизнеса)
H2: Пошаговый план
H2: Типичные ошибки
H2: FAQ (3–4 вопроса)
[Ссылки на все затронутые инструкции]
```

**Тег:** `кейс`, плюс теги по кластерам.

---

### 4. Новостная заметка (300–500 слов)

Реакция на инфоповод: новый НПА, разъяснение ФНС, изменение сроков.

**Когда публиковать:** в течение 24–48 часов после выхода документа.

**Структура:**
```
H1: [Что изменилось] — [дата/источник]
Лид: суть изменения одним абзацем
H2: Что конкретно меняется (было → стало)
H2: Кого касается
H2: Что делать прямо сейчас
[Ссылка на первоисточник (НПА) + внутренние ссылки]
```

**Тег:** `новости`, `обновление`.

**Параллельно:** обновить `updatedDate` в связанных статьях кластера.

---

## Шаблоны документов

Хранятся в `/public/templates/`. Ссылка на скачивание прямо из статьи:

```markdown
[Скачать шаблон приказа о внедрении КЭДО (.docx)](/templates/prikaz-kedo.docx)
```

**Правило:** публиковать шаблоны только с свободной формой (не утверждённой).
Если форма утверждена НПА — давать ссылку на официальный источник, не копию.

Текущие шаблоны:
- `prikaz-kedo.md` — приказ о внедрении кадрового ЭДО (свободная форма)

---

## Процесс обновления старых статей

1. Мониторинг: routine раз в 3 дня (см. `wiki/routine-prompt.md`)
2. При изменении НПА: обновить `updatedDate`, исправить цифры/сроки
3. Если изменения существенные: написать новостную заметку + дать ссылку из обновлённой статьи
4. Проверить: `grep -rn "старая норма" src/content/blog/` — нет ли упоминаний в других постах

---

## Приоритеты публикации

| Тип | Срок реакции | Инструмент |
|---|---|---|
| Инфоповод (новый НПА, штрафы) | 24–48 часов | Написать напрямую |
| P0 из контент-плана | Текущая неделя | `/new-post` |
| Быстрый ответ | 1–2 дня | Написать напрямую |
| Кейс | Текущая неделя | `/new-post` или напрямую |
| P1 из контент-плана | Следующий месяц | `/new-post` |
