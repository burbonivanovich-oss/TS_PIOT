Вывод из оборота — операция, которой маркированный товар выходит из системы прослеживаемости «Честный знак». Для розничного магазина вывод из оборота происходит в момент продажи через кассу. С введением разрешительного режима этот процесс жёстко привязан к ТС ПИоТ.

## Что такое вывод из оборота

После того как товар с кодом DataMatrix был произведён или ввезён, он числится в системе «Честный знак» как «в обороте». Каждая передача между участниками цепочки (производитель → дистрибьютор → магазин) отражается в системе через ЭДО. Когда товар продаётся конечному потребителю — он выводится из оборота: код аннулируется, товар больше не числится в цепочке прослеживаемости.

Основания для вывода из оборота:
- Продажа через кассу (основной способ в рознице).
- Использование в производстве.
- Утилизация или уничтожение.
- Конфискация.
- Экспорт.

## Как происходит вывод через кассу

При продаже маркированного товара через онлайн-кассу процесс выглядит следующим образом:

1. Кассир сканирует код DataMatrix на упаковке.
2. Кассовое ПО передаёт код в ТС ПИоТ.
3. ТС ПИоТ направляет запрос в «Честный знак» и получает ответ: «разрешено» или «запрещено».
4. При положительном ответе — продажа проходит, код записывается в фискальный чек (ФФД 1.2).
5. ФНС через ОФД получает данные о продаже.
6. «Честный знак» автоматически выводит код из оборота на основании данных чека.

С 1 июля 2026 года шаги 2–3 обязательны для всех торговых точек, продающих маркированные товары (ст. 4.3 54-ФЗ). Подробнее о ТС ПИоТ — в статье [что такое ТС ПИоТ](/blog/chto-takoe-ts-piot).

## Разрешительный режим: что меняется

Разрешительный режим — более строгий вариант работы, при котором касса физически не может завершить продажу без получения разрешения от «Честного знака». Подробно механизм описан в материале [разрешительный режим маркировки](/blog/razreshitelnyj-rezhim-markirovka).

При разрешительном режиме:
- Касса блокирует продажу товара с кодом «запрещён к продаже» (просроченная маркировка, изъятый из оборота товар, повторное использование кода).
- Продажа без сканирования невозможна технически.

Разрешительный режим введён для молочной продукции, табака, обуви, одежды и ряда других категорий; в 2026 году распространится на новые группы товаров.

## Как настроить кассу для вывода из оборота

**Шаг 1. Убедиться, что касса поддерживает ФФД 1.2.**
Форматы фискальных документов версии 1.2 позволяют включить в чек коды маркировки. Без ФФД 1.2 корректный вывод из оборота невозможен. Касса обновляется через производителя или сервисный центр.

**Шаг 2. Активировать ТС ПИоТ.**
ТС ПИоТ — программный модуль. Для разных кассовых платформ он устанавливается по-разному. У Эвотора — через магазин приложений. У АТОЛ и других — через кассовое ПО или отдельное приложение. Процедура подключения — в статье [подключение ТС ПИоТ](/blog/ts-piot-podklyuchenie-instrukciya).

**Шаг 3. Настроить сканер DataMatrix.**
Сканер должен читать коды формата DataMatrix (2D). Большинство современных 2D-сканеров совместимо. Убедитесь, что сканер передаёт код в кассовое ПО без лишних символов — иначе ТС ПИоТ не сможет идентифицировать товар.

**Шаг 4. Протестировать продажу маркированного товара.**
Проведите тестовую продажу с реальным кодом DataMatrix. Убедитесь, что в чеке содержится код маркировки (раздел «Предмет расчёта» чека по ФФД 1.2).

## Типичные ошибки при выводе из оборота

**Касса передаёт данные в ОФД, но не в «Честный знак».** Это случается, если ТС ПИоТ не установлен или отключён. Код в чеке есть, но вывод из оборота не происходит — товар продолжает числиться в системе. Это нарушение.

**Двойной вывод из оборота.** Повторное сканирование и продажа кода, который уже выведен, — система заблокирует операцию. Если ошибочно отсканировали один и тот же код дважды — нужно отменить чек и оформить заново.

**Возврат без восстановления в обороте.** При возврате товара его код должен быть возвращён в оборот. Это выполняется через кассовое ПО при оформлении чека возврата с кодом DataMatrix.

## Особенности при объёмно-сортовом учёте

При объёмно-сортовом учёте (молоко, напитки, консервы) в чек включается код DataMatrix каждой единицы, хотя в УПД при приёмке коды могли не перечисляться. Вывод из оборота через кассу происходит поединично в любом случае.

Разницу между объёмно-сортовым и поэкземплярным учётом описывает статья [объёмно-сортовой и поэкземплярный учёт](/blog/obyemno-sortovoy-vs-poexzemplyarnyj).

## FAQ

**Что делать, если код DataMatrix не считывается сканером?**
Попробуйте другой угол сканирования. Если код повреждён — товар нельзя продать стандартным способом. Нужно обратиться к поставщику за перемаркировкой или провести перемаркировку самостоятельно.

**Нужно ли выводить из оборота товар при списании?**
Да. Товар, который не будет продан (испорчен, утилизирован), выводится из оборота в «Честном знаке» с указанием причины «утилизация». Без этого он продолжает числиться в вашем остатке.

**Можно ли продать маркированный товар без кассы (например, самозанятому)?**
Нет. Продажа маркированных товаров физическим лицам — конечным потребителям — должна сопровождаться кассовым чеком с кодом маркировки.

**Как оформить вывод из оборота при краже?**
Через личный кабинет «Честного знака» с указанием основания «утрата / хищение». Коды, выведенные по этому основанию, нельзя в дальнейшем использовать в продаже.

**Что будет, если не выводить из оборота проданный товар?**
Инспекция «Честного знака» при сопоставлении данных ОФД и системы прослеживаемости выявит несоответствие. Это квалифицируется как нарушение порядка оборота маркированных товаров с последующими санкциями.

При правильной настройке вывод происходит автоматически при каждой продаже.
