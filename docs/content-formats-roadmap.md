# Roadmap: новые контент-форматы (#33)

Стратегическое направление — уйти от базовой информационки в форматы,
которые сложнее повторить конкурентам. Дискуссия 2026-05-19 свела
25 идей в шорт-лист и план из 3 спринтов.

> **Главная инновация:** F26 — интерактивные блоки внутри статей.
> Это не отдельный формат, а **слой над всеми остальными**.
> Вдохновение: [The Internet of Trust](https://notdotteam.github.io/trust),
> [Markov chains explainer](https://molyanov.ru/blog/markov-chains).

> **Итерация 2 (2026-05-19, вечер).** Перечитав примеры, поняли: мы
> недооценивали амбицию. Notdotteam и Молянов делают не «статья +
> калькулятор», а **статья, где интерактив и есть объяснение** — без
> него её нельзя прочитать так же глубоко. Сместили F26-7 (пошаговый
> симулятор ТС ПИоТ как пример Markov) из Спринта C в Спринт A как

## F26 — почему сквозной слой

Налоги, маркировка, ккт — это **системы с правилами**. Правила
естественно превращаются в интерактив:

| Тема | Интерактивный блок |
|---|---|
| УСН-доходы vs Доходы-расходы | Ползунок «оборот: ___» → меняются итоговые налоги |
| ТС ПИоТ дедлайн | Слайдер времени → что должно быть сделано |
| Маркировка по категории | Селектор «Что продаёте?» → таймлайн дат |
| Штраф за немаркировку | Радио «Юрлицо/ИП» + слайдер выручки → итог + ст. КоАП |
| Срок ФН | Тогл «ОСН/УСН» + «Подакцизные/Нет» → ФН 15/36 мес |
| Передача данных ТС ПИоТ | Сценарий-симулятор по шагам (как пример Markov) |
| Цепочка ЭДО при маркировке | Drag-and-drop собрать последовательность |
| Оборотный штраф | Калькулятор «Выручка год → штраф = X% от Y» |

**Эффект** (по аналогии с notdotteam.github.io/trust):
- Время на странице: +60–120%
- Глубина прокрутки: +30%
- Отстройка: максимальная (никто из конкурентов так не делает)
- Цитируемость AI: средняя (текст вокруг становится лучше)

## Технический подход

У нас Astro 5 + MDX уже включён (`content.config.ts` принимает
`*.{md,mdx}`).

```mdx
import UsnCalc from '/src/components/interactive/UsnCalc.astro';

## Как считать УСН с НДС

Базовая ставка УСН-доходы — 6%. Но с 2026 года, если оборот
превышает 20 млн ₽, добавляется НДС.

<UsnCalc client:load />

Введите оборот выше — увидите налог при разных режимах.
```

Каждый интерактив — Astro/React-компонент в
`src/components/interactive/`. Размер 50–150 строк. 2–4 часа на
компонент.

**Аналитика интерактивов** через Метрику-цели:

```ts
// src/utils/track.ts
export function track(event: string) {
  if (typeof window === 'undefined') return;
  window.ym?.(109130279, 'reachGoal', event);
}
```

Используется в каждом компоненте при ключевых действиях
(`reachGoal('usn-calc-used')`).

## Спринт A — флагман-симулятор + журнал + первые калькуляторы (3 недели, ~40 ч)

> Расширили с 2 недель/26 ч до 3 недель/40 ч из-за F26-7-флагмана.
> Это правильный объём: один сильный материал важнее трёх слабых.

### Артефакты

1. **F26-7 ФЛАГМАН: `/kak-rabotaet-ts-piot/`** — пошаговый
   симулятор «Как живёт чек в ТС ПИоТ» (касса → ОФД → ФНС → ТС ПИоТ
   → ЧЗ). Самостоятельный лонгрид со сценарием 5–7 шагов: каждый шаг
   — иллюстрация + объяснение + кнопка «дальше» + ветка «а что если
   ошибка?». Прямая аналогия с примером Markov. Сценарий —
   `src/content/wiki/flagship-ts-piot-simulator.md`. Без сценария
   код не пишем.
2. **`/zakon-2026/`** (F13) — хронологический трекер изменений.
   Astro-страница с таймлайном: дата + норма + 1 предложение +
   ссылка на наш разбор.
3. **F26-1: фильтр-таймлайн на F13.** Ползунок «месяц» → видны
   только нормы, вступающие в этом окне.
4. **F26-2: калькулятор НДС/УСН в `nalogi-msb-2026`** *(вторичный)*.
   Ввод оборота → УСН 6%/15% + ставки НДС 5/7/22%.
5. **F26-3: слайдер штрафа в `shtraf-za-markirovku`** *(вторичный)*.
   Ползунок «выручка партии» + переключатель «ИП/Юрлицо» → итог
   штрафа + ст. КоАП.

### Метрики

- **Флагман:** ≥ 300 уникальных «прошёл до конца» в `/kak-rabotaet-ts-piot/`
  за первый месяц, среднее время на странице ≥ 4 минут, доля доскрола
  до последнего шага ≥ 40%.
- Прирост позиции в GSC/Метрике по запросам «как работает тс пиот»,
  «передача данных в честный знак».
- Цитируемость в Yandex Neuro / ChatGPT Search (по выборочным проверкам).
- По вторичным калькуляторам: ≥ 100 уникальных взаимодействий/мес.

### Что нужно перед стартом

- **Сценарий флагмана** в `src/content/wiki/flagship-ts-piot-simulator.md` —
  делается до кода. Согласовать структуру, визуальный язык, тексты шагов.
- Папка `src/components/interactive/`
- `src/utils/track.ts` с обёрткой `track()` под Метрику
- Цели в Яндекс.Метрике: по каждому шагу флагмана `step-N-reached`
  (для воронки доскрола) + по 1 цели на калькулятор.
- Проверка работы MDX-импортов в `src/content/blog/` и для отдельной
  страницы-лонгрида (`src/pages/kak-rabotaet-ts-piot.astro` или
  `.mdx` через [...slug]).
- Решение по визуалу шагов: генерим через OpenRouter + Nano Banana
  (как hero) или ручные SVG-схемы.
