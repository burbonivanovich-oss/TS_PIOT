# Factcheck — проверка фактов в опубликованных статьях

Свой стек проверки claims (дат, штрафов, ссылок на НПА) с автоматическим
извлечением и ручной/агентной верификацией через WebSearch и WebFetch.

## Зачем

Контент про регулирование МСБ устаревает быстро — сроки маркировки,
суммы штрафов в КоАП, постановления правительства меняются часто. Без
проверки статья 6-месячной давности легко содержит неактуальные цифры.

## Структура

```
scripts/factcheck/
  extract-claims.mjs       # regex-парсер: даты, суммы, ст. КоАП, № ФЗ, ссылки
                            # запуск: node scripts/factcheck/extract-claims.mjs <slug>
                            #        node scripts/factcheck/extract-claims.mjs --all

src/data/factcheck/
  sources.json             # маппинг типов claims на первоисточники +
                            # список доменов, заблокированных для WebFetch
  inventory.json           # сводка claims по всем 83 статьям (с --all)
  claims/<slug>.json       # claims одной статьи
  results/<slug>.json      # отчёт проверки: status, severity, action

.claude/
  commands/factcheck.md    # /factcheck <slug> — скилл проверки
  factchecked/<slug>       # маркер «статья проверена» (файл с датой)
```

## Поддерживаемые типы claims

| Тип | Что ловит | Пример |
|---|---|---|
| `DATE_DMY` | дата в формате DD.MM.YYYY | `01.07.2026` |
| `DATE_TEXT` | дата словами | `28 декабря 2025 года` |
| `DATE_YEAR` | год с суффиксом | `2026 году` |
| `DATE_RANGE` | диапазон лет | `2024–2026` |
| `MONEY` | денежная сумма | `15 000 ₽`, `1 млн руб.` |
| `NPA_KOAP` | статья КоАП РФ | `ст. 15.12 КоАП`, `ст. 14.5 ч. 4 КоАП` |
| `NPA_FZ` | номер федерального закона | `54-ФЗ`, `289-ФЗ` |
| `NPA_FZ_FULL` | полная формулировка | `Федеральный закон от 01.04.2020 № 102-ФЗ` |
| `PERCENT` | процент | `20%`, `7,5 %` |
| `LINK` | внешняя ссылка | URL из markdown |

Регулярки лежат в `scripts/factcheck/extract-claims.mjs`. Если паттерн не
ловится — добавьте новый и регенерируйте `inventory.json`.

> **Важно:** `\b` под флагом `u` не работает с кириллицей (Cyrillic не
> считается word-char в Unicode mode). Для русских паттернов используем
> `(?<!\d)` / `(?!\d)` и явные ASCII-границы.

### Заблокированные домены

`consultant.ru`, `pravo.gov.ru`, `yandex.*` отдают **403 WebFetch** (anti-bot).
Для них используем `WebSearch` — он возвращает сниппеты с релевантными
цифрами. Скилл `/factcheck` сам выбирает WebFetch или WebSearch на основе
`blocked.domains` из sources.json.

## Цикл проверки

```
/factcheck <slug>
   │
   ├── 1. node extract-claims.mjs <slug>     # claims/<slug>.json
   │
   ├── 2. для каждого claim:
   │     ├── lookup в sources.json
   │     ├── если домен в blocked → WebSearch с тематикой+claim
   │     └── иначе → WebFetch с целевым запросом
   │
   ├── 3. сравнить ответ с claim:
   │     ├── match — confidence 0.9+, ничего не делаем
   │     ├── mismatch — расходится, severity critical/moderate/minor
   │     ├── uncertain — не понятно, нужен ручной просмотр
   │     └── missing — нет первоисточника в статье (например, нет ссылки)
   │
   ├── 4. записать results/<slug>.json
   │
   ├── 5. поставить маркер .claude/factchecked/<slug>
   │
   └── 6. вывести summary в чат:
         🔴 critical N → блокировка публикации, draft: true
         🟡 moderate N → правки
         🟢 match N
```

## Severity и action

| Severity | Когда применять | Action |
|---|---|---|
| `critical` | НПА изменился, дата неверна, сумма штрафа отличается ×2+ | Откатить в `draft: true` до правки |
| `moderate` | Часть статьи не указана, формулировка устарела, нет полного контекста | Правки в текущей редакции |
| `minor` | Опечатка, форматирование, лишний предлог | Можно отложить до планового update |

`action` — конкретный тип правки: `rewrite-lede`, `rewrite-bullet`,
`expand-bullet`, `add-references`, `add-disclaimer`, `keep`. В поле
`actionDetail` — готовый текст замены, чтобы не возиться с
переформулировкой.

## Маркетинговый баланс

Не каждое расхождение требует переписывать под букву первоисточника.
Контент коммерческий — продаёт CPA-баннеры. Различаем:

- **Класс A — точность важнее подачи.** Суммы штрафов, номера ст. КоАП,
  даты НПА, технические требования. Здесь `mismatch` = критика, переписать.
- **Класс B — маркетинговая формулировка.** Дедлайн как «дедлайн перехода»
  вместо «дата начала действия» — оправдано для создания срочности. Здесь
  не выкидываем формулировку, а **добавляем disclaimer-блок** с полной
  картиной (`status: incomplete-context`, `action: add-disclaimer`).

Пример (из пилота): «до 1 июля 2026» в лиде остаётся как маркетинг-хук
(«осталось несколько месяцев — успейте»). В blockquote сразу под лидом —
«формально с 28.12.2025 (ПП РФ № 1674), до 1.07.2026 разрешена X-API-KEY,
далее только ТС ПИоТ». И срочность работает, и факт честный.

## Когда extract-claims не ловит claim

- Числа словами («пятнадцать тысяч рублей») — не покрыто.
- Контекстные даты («с начала года», «через месяц») — не покрыто.
- Пункты НПА без «ст.» («п. 1.2 ПП РФ № 1674») — не покрыто.

Это нормально: фактчек — первый фильтр, а не замена редактору. Если в статье
важные claims «прошли мимо» — добавляйте паттерн и регенерируйте `inventory.json`.
