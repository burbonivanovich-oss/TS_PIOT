# Roadmap блоков сайта

Документ описывает, какие блоки используются сейчас, какие отобраны
из лабораторных прототипов (`prototypes/homepage-blocks-lab*.html`,
каталог — `prototypes/homepage-catalog-full.html`) и куда они идут.

Источник идей: 124 эксперимента в 6 lab-файлах. Из них пользователь
отобрал 27 в эту дорожную карту.

---

## 2. Отобранные блоки из лабораторий

Формат: **код · название** → где использовать → с чем конкурирует →
решение.

### 2.1 Главная страница

#### LAB4 #1 — Asymmetric Bento
**Что это.** Сетка из крупного и двух мелких блоков разной массы:
один большой featured + два меньших + ряд иконок снизу.

- **Где использовать:** заменить нынешний блок «Последние материалы»
  на главной. Один свежий лонгрид крупно + 2 свежих средних + ряд из
  4–6 миниатюр.
- **Конкурирует с:** H4 (Latest posts — featured + 6 равных карточек).
- **Решение:** **заменить**. Текущая сетка 3×n визуально плоская,
  bento добавляет иерархию без новых сущностей.

#### LAB3 #11 — Image-led grid
**Что это.** Сетка, где картинка занимает 70% карточки, текст —
снизу узкой полосой. Фото-first вёрстка.

- **Где использовать:** `/blog/` и `/category/[slug]/`. Опционально —
  как альтернативный режим карточек в `BlogFilter.tsx`.
- **Конкурирует с:** текущая карточка статьи (`docs/article-cards.md`)
  с пропорцией 16:9 и одинаковой высотой текста.
- **Решение:** **добавить как опциональный режим**. Текущий формат
  работает на главной (картинки качественные, описание помогает),
  но в `/blog/` при 30+ карточках image-led эстетичнее.

#### LAB1 #5 — Long-read spotlight (Guardian-style)
**Что это.** Один материал на всю ширину: огромный заголовок,
крупное фото, лид-абзац, eyebrow «Лонгрид · 12 мин».

- **Где использовать:** на главной выше «Последних материалов» — для
  свежего pillar или флагманского сценария.
- **Конкурирует с:** featured-карточка в H4. По сути это её
  усиленная версия.
- **Решение:** **заменить featured** на Long-read spotlight тогда,
  когда верхний материал — pillar (>2500 слов) или флагман.
  В остальные дни — обычная featured-карточка.

#### LAB1 #1 — Magazine combo (облегчённый)
**Что это.** Журнальная сетка: левая колонка с крупным материалом,
правая — список из 3–5 рубричных подборок.

- **Где использовать:** альтернатива H4 для дней, когда вышло 5+
  свежих статей. Помогает показать «много нового» без длинной ленты.
- **Конкурирует с:** H4 + H2 одновременно.
- **Решение:** **доработать и держать в запасе**. Текущая версия
  перегружена — нужно сократить до 3 рубрик справа максимум.
  Применять не постоянно, а как сезонный лейаут (после публикаций
  пачкой).

#### LAB3 #4 — Series / cycle
**Что это.** Блок «Серия из N статей»: первая крупная + остальные
карточки в ряд с нумерацией 1/4, 2/4, 3/4, 4/4.

- **Где использовать:** на главной как третий блок после Hero и
  Bento, когда есть активный pillar-кластер (например, 140-ФЗ
  ЭПД + 6 сателлитов).
- **Конкурирует с:** нет прямого конкурента. Это новый тип блока.
- **Решение:** **добавить**. Pillar-кластеры — главная стратегия
  кластеризации трафика, но сейчас они никак не видны на главной.

#### LAB3 #13 — Category section header
**Что это.** Большая цветная плашка-разделитель с названием категории
и 3–4 карточками статей под ней.

- **Где использовать:** на главной — после Hero, чтобы внутри одной
  ленты «Последние материалы» сегментировать материалы по категориям.
  По 1 плашке на категорию подряд: ТС ПИоТ → 3 статьи → Маркировка →
  3 статьи → Законы → 3 статьи.
- **Конкурирует с:** H2 (Category tiles) — обе подсвечивают категории.
- **Решение:** **держать обе**. H2 — навигационный быстрый вход
  в раздел (5 секунд на сканирование), Section header — погружение
  в свежак конкретной категории.

#### LAB1 #18 — CTA band
**Что это.** Широкая партнёрская плашка во всю ширину: лого
партнёра + оффер + кнопка.

- **Где использовать:** на главной между Tools strip и Latest posts.
  Один баннер за раз, ротация по категории партнёра.
- **Конкурирует с:** `PopupBanner.tsx` (всплывающий) + inline
  `CpaCallout` внутри статей.
- **Решение:** **добавить как третий канал**. Popup — для возвратных
  пользователей; inline — внутри статьи; CTA band — для тех, кто
  пришёл на главную с поиска и до статьи не дошёл. **Обязательно**
  с пометкой «Реклама» + erid.

#### LAB4 #18 — Promoted с пометкой «Реклама»
**Что это.** Карточка статьи в общей ленте, но с явным фоном-плашкой
«Реклама», erid и логотипом партнёра.

- **Где использовать:** внутри Latest posts на главной и `/blog/`.
  Раз в 8 органических карточек — одна promoted.
- **Конкурирует с:** CTA band (LAB1 #18) и `CpaCallout`. У Promoted
  ниже плотность и она органичнее встроена в ленту.
- **Решение:** **добавить, но позже**. Сначала — Bento + CTA band,
  потом, когда поток партнёрских материалов пойдёт стабильно
  (4+ креатива на категорию), включить Promoted в ленту.

#### LAB4 #16 — Mini-newsletter inline
**Что это.** Узкая полоса «Подпишитесь на рассылку» с инпутом email,
без модалки и оверлеев.

- **Где использовать:** на главной между Latest posts и Footer, и
  опционально внутри статьи между 5-м и 6-м абзацем.
- **Конкурирует с:** `PopupBanner.tsx`.
- **Решение:** **заменить popup** для возвратных пользователей,
  оставить popup только для первой сессии. Inline-форма даёт
  конверсию на хедхантерах ~3×, не раздражает.

#### LAB6 #18 — Live countdown
**Что это.** Тикающий счётчик до даты («До 01.07.2026 осталось
36 дней 14 часов»). Обновляется в реальном времени.

- **Где использовать:** баннер сверху главной (под Header) — отсчёт
  до ключевых дат: ТС ПИоТ обязателен, очередная волна маркировки,
  140-ФЗ переход. По 1 счётчику за раз, ротация по приоритету.
- **Конкурирует с:** ничем — это новая сущность.
- **Решение:** **добавить**. Соответствует tone of voice
  («требования догоняют бизнес»), создаёт ощущение срочности без
  кликбейта.

#### LAB3 #7 — Bottom-line digest «суть в 1 строке»
**Что это.** Плотный список заголовков с однострочным резюме
после каждого («что произошло» в одном предложении).

- **Где использовать:** на странице `/blog/` как альтернативный
  режим отображения. Также — внутри Category section header
  (LAB3 #13) для «остальных материалов раздела».
- **Конкурирует с:** карточки в `BlogFilter.tsx`.
- **Решение:** **добавить как переключатель режима** на `/blog/`:
  «Карточки / Дайджест». Дайджест для тех, кто читает RSS-стилем.

### 2.2 Глоссарий

#### LAB2 #15 — Glossary A-Я
**Что это.** Сетка из плиток с буквами А-Я. На каждой — счётчик
терминов; ховером раскрывается превью первых 3 терминов.

- **Где использовать:** заменить текущий алфавитный индекс на
  `/slovar/`.
- **Конкурирует с:** нынешний линейный алфавит в `/slovar/`.
- **Решение:** **заменить**. Текущая версия — простой список;
  плиточная сетка с превью и счётчиком даёт быстрый вход
  «куда копать».

### 2.3 Инструменты — новые

#### LAB1 #2 — Quiz «подбор сервиса под бизнес»
**Что это.** 4–5 шагов: «Что у вас?» (категория) → «Сколько чеков?»
(размер) → «Что важно?» (фичи) → рекомендация сервиса/инструмента
с CPA-ссылкой.

- **Где использовать:** новый раздел `/podbor/` или `/quiz-podbor/`.
  Кнопка-вход в Hero на главной (заменит текущую кнопку `Quiz`).
- **Конкурирует с:** `Quiz.tsx` (TsPiotReadinessQuiz) — тест
  готовности к ТС ПИоТ. Они разного назначения: текущий проверяет
  знание правил, новый помогает выбрать сервис.
- **Решение:** **добавить параллельно**. На главной — две кнопки
  («Проверить готовность» + «Подобрать сервис»). Партнёрские
  ссылки в рекомендациях — критичный путь монетизации.

#### LAB1 #12 — Comparison table — сравнение касс/сервисов
**Что это.** Большая таблица сравнения: ряды — продукты, колонки —
характеристики, last row — кнопки CTA. Sticky-заголовок.

- **Где использовать:** новый раздел `/sravneniya/` или
  `/kassy-sravnenie/`, `/servisy-edo-sravnenie/` и т. п. По одной
  странице на категорию сравнения.
- **Конкурирует с:** статьи-сравнения в `src/content/blog/`
  (например, «Атол vs МТС Касса»).
- **Решение:** **добавить как отдельный формат** F-27. Статьи дают
  контекст и SEO, отдельная страница со sticky-таблицей —
  конверсионный инструмент. Перелинковка: из статьи → в таблицу.

#### LAB2 #3 — Cost slider — интерактивный расчёт
**Что это.** Слайдер «количество чеков в месяц» / «выручка» → в
реальном времени считает стоимость кассы/тарифа/штрафа.

- **Где использовать:** встраивать F26-блоком в статьи о тарифах
  и стоимости (например, «сколько стоит ТС ПИоТ для кофейни»).
  Также — рядом с calculator-страницами.
- **Конкурирует с:** `PenaltyCalculator.tsx` — там сценарии,
  а тут одна переменная.
- **Решение:** **добавить как F26-блок**. Не заменяет калькулятор
  штрафов (тот сложнее), но даёт быстрый эффект «крути и смотри»
  внутри статей.

#### LAB2 #14 — Big toggle question
**Что это.** Большой да/нет вопрос на весь экран: «Вы работаете
с маркируемым товаром?» → ДА/НЕТ → дальше разный контент.

- **Где использовать:** редко. На лендингах конкретных категорий
  (`/markirovka/start/`) или в начале сложных гайдов как gate.
- **Конкурирует с:** длинные интро в статьях.
- **Решение:** **держать в запасе для лендингов**. Не для главной
  и не для блога, но идеально для одиночных landing pages под
  конкретный таргет.

#### LAB2 #16 — Wizard multistep preview
**Что это.** Видимая дорожка шагов квиза с активным шагом, чтобы
пользователь видел прогресс заранее.

- **Где использовать:** надстройка над всеми текущими квизами
  (`Quiz.tsx`, тест ТС ПИоТ, тест ЭДО) и над новым LAB1 #2.
- **Конкурирует с:** текущие квизы показывают «шаг 3 из 5» строкой.
- **Решение:** **обновить все квизы**. Видимая дорожка снижает
  drop-off — пользователь понимает, сколько осталось.

#### LAB6 #12 — Diagnostic quiz
**Что это.** Квиз-диагностика без правильных ответов: 7–10 вопросов,
на выходе — карта рисков «у вас 4 красных / 2 жёлтых / 1 зелёный».

- **Где использовать:** **не для подбора сервиса**, а как «аудит»:
  «Проверьте, нет ли у вас нарушений по маркировке» (категории
  риска ст. 15.12 КоАП). Конверсия: «Скачать чеклист исправления».
- **Конкурирует с:** `Quiz.tsx` (там pass/fail логика).
- **Решение:** **добавить как F26 «Аудит-квиз»**. По одному
  на категорию — маркировка, ЭДО, кассы.

#### LAB6 #8 — AI consultant (с RAG)
**Что это.** Чат-инпут «спросите эксперта». Ответы из RAG-базы
собственных статей + НПА.

- **Где использовать:** новая страница `/assistant/` + плавающая
  кнопка в углу главной.
- **Конкурирует с:** `/search/` (Pagefind).
- **Решение:** **исследовать перед внедрением**. Требует:
  (а) RAG-pipeline с эмбеддингами всех статей, (б) контроль
  галлюцинаций (модель не должна выдумывать штрафы), (в) бюджет
  на API. Pagefind остаётся как fallback. Заложить в roadmap
  Q3 2026 после стабилизации контент-плана.

#### LAB6 #20 — Live Q&A board
**Что это.** Доска вопросов: пользователь оставляет вопрос →
ответ публикуется (от редакции или RAG-бота). Хронологическая
лента.

- **Где использовать:** отдельная страница `/voprosy/` — на
  будущее, после AI consultant.
- **Конкурирует с:** комментарии (которых нет) + AI consultant.
- **Решение:** **на будущее**. Сначала AI consultant —
  это асинхронный «ответ сразу». Board — социальное доказательство
  («сотни предпринимателей уже спросили»), включать после набора
  массы вопросов.

#### LAB1 #7 — Roadmap (Coursera-стиль)
**Что это.** Персональный маршрут «прочитайте 8 статей в таком
порядке, чтобы разобраться в ТС ПИоТ». Прогресс-бар + галочки.

- **Где использовать:** новый раздел `/marshrut/` или подраздел
  внутри каждого pillar. Маршрут на pillar — 5–8 статей по
  возрастанию сложности.
- **Конкурирует с:** «Читайте также» в `BlogPost.astro` и текущая
  плитка категорий.
- **Решение:** **добавить точечно** под 2–3 ключевых pillar
  (ТС ПИоТ, Маркировка с нуля, 140-ФЗ). Не для каждой статьи.

#### LAB1 #6 — Streak (Duolingo)
**Что это.** Геймификация чтения: «вы читаете 3 дня подряд»,
бэйджи за серии.

- **Где использовать:** в правом верхнем углу для возвратных
  пользователей (LocalStorage).
- **Конкурирует с:** ничем.
- **Решение:** **попробовать**, но низкий приоритет. Без
  регистрации работает только на стабильных возвратах (с одного
  устройства). Можно делать как «лайт»-версию: «прочитайте 3
  статьи и получите PDF-чеклист».

#### LAB1 #10 — Author spotlight (NYT-стиль)
**Что это.** Карточка автора: фото, био, его статьи, кнопка
«подписаться на автора».

- **Где использовать:** не для главной. Подходит для
  `/blog/[slug]/` (под статьёй) и для будущей страницы
  `/avtory/<name>/`.
- **Конкурирует с:** ничем (сейчас у нас нет автора в карточке).
- **Решение:** **внедрить, если выйдет внешний экспертный
  автор-партнёр** (юрист, бухгалтер, владелец сети касс). Для
  редакционного контента «Этикетка» — не имеет смысла, аудитория
  читает бренд, а не персону.

### 2.4 Внутри статей (F26-блоки)

#### LAB5 #1 — Q&A Dialog
**Что это.** Внутри статьи — диалоговый разворот: «Вопрос
читателя: …» → «Ответ редакции: …». Графически отделён от prose.

- **Где использовать:** F26-блок внутри статей-разборов сложных
  норм. По 1–2 за статью.
- **Конкурирует с:** `FAQ.astro` (FAQ внизу статьи).
- **Решение:** **добавить как новый F26-блок**. FAQ — финальный
  блок «горячих вопросов»; Q&A Dialog — нативно встроенный в
  повествование («тут читатель часто спрашивает X — отвечаем»).

#### LAB3 #18 — Download collection — сборник в PDF
**Что это.** Карточка «Скачать сборник» с превью обложки,
содержанием и кнопкой. Email-gated или открыто.

- **Где использовать:** на pillar-страницах после прохождения
  diagnostic-квиза (LAB6 #12) и в `CpaCallout`-местах. По 1–2
  PDF на pillar.
- **Конкурирует с:** `CpaCallout` (текущий CPA-блок).
- **Решение:** **добавить как новый тип CTA**. CpaCallout —
  партнёрский (платная конверсия), Download collection — лидмагнит
  (наша подписка). Разделить две воронки.

### 2.5 Спец-проекты / отдельные статьи

#### LAB5 #17 — Pinboard scrapbook
**Что это.** Доска заметок с вырезками, post-it-стикерами,
скрепками, фото. Художественный формат.

- **Где использовать:** только спец-проекты. Например, «Год в
  жизни бухгалтера МСБ» — лонгрид-ретроспектива со скриншотами
  ЭДО, цитатами, фрагментами законов как «вырезки».
- **Конкурирует с:** ничем — это уникальный формат.
- **Решение:** **держать как формат для 1–2 спецпроектов в год**.
  Дорого делать, но даёт виральность и брендинг.

#### LAB5 #18 — Card game deck
**Что это.** Колода карточек, листаются как карты. У каждой —
«ход», «штраф», «совет». Имитация настолки.

- **Где использовать:** спец-проект «Карты выживания МСБ» —
  колода по типам проверок и нарушений. Или ежегодный «Адвент-
  календарь предпринимателя» (25 карт-советов в декабре).
- **Конкурирует с:** ничем.
- **Решение:** **один спец-проект в год**. Адвент-календарь
  декабрь 2026 — хороший таргет.

## 4. Что НЕ берём (явный отказ)

Чтобы не возвращаться к 124 идеям повторно — фиксируем причины
отказа от блоков, которые рассматривали:

- **B2C-эксцентрика** (dice roll, swiper-карты, heatmap-контрибы,
  big quote-полотно) — не соответствует B2B-tone of voice.
- **Stories-полоса** (LAB4 #5) — мимикрия под Instagram неуместна
  в media для МСБ.
- **Sankey / word cloud / sticker pack** — визуально броско, но
  не помогает конверсии и сложно поддерживать.
- **Spotify-carousel** (LAB1 #19) — карусель проигрывает простой
  сетке по навигабельности.
