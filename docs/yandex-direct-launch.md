# Запуск РК в Яндекс.Директ — готовый пакет

Полный набор для старта платной кампании etiketka-media.ru.
Семантика — на реальной частотности из `src/data/wordstat/keys.json` (Wordstat)
и фактических запросов GSC. Дата: 2026-06-20.

---

## 0. Экономика и осторожность (прочитать первым)

Сайт зарабатывает на **CPA Контура** (заявки/лиды), а не на прямых продажах.
Поэтому платный трафик окупается, только если:

```
доход с лида (CPA) × конверсия клик→лид  >  цена клика (CPC)
```

Пример: CPC 30 ₽, конверсия клика в заявку 3% → стоимость лида 1000 ₽.
Если CPA Контура за лид/сделку выше — плюс; ниже — минус.

**Вывод:** стартуем **минимальным тестом** (30 000–50 000 ₽/мес), только на
**горячих коммерческих запросах** с посадкой на витрину/страницу продукта
(где лид-форма), мерим стоимость лида по цели `lead-*` и **масштабируем
только прибыльные группы**. Информационные ВЧ-запросы («честный знак»
1,3 млн, «онлайн-касса» 84 тыс) на старте НЕ берём — сольют бюджет.

## 2. Общие настройки кампаний

- **Регион:** Россия (минус Москва/СПб на старте можно НЕ ставить — но следить
  за CPC; при дорогом клике сузить до регионов с дешёвым трафиком).
- **Стратегия:** старт — «Максимум кликов с ручным управлением» или
  «Оплата за конверсии» (цель `lead-*`), когда накопится ≥10 конверсий —
  «Максимум конверсий» с целевой ценой лида.
- **Бюджет:** K1 — 1500–2500 ₽/день. Общий тест — 30–50 тыс ₽/мес.
- **Расписание:** круглосуточно или 8:00–22:00 МСК (B2B-активность днём).
- **Площадки (для РСЯ K4):** отключить мусорные, минус-площадки добавлять
  по отчёту еженедельно.
- **Корректировки ставок:** +0% мобильные на старте (потом по данным),
  целевая аудитория — ретаргетинг посетителей.
- **Расширения:** включить быстрые ссылки, уточнения, отображаемую ссылку,
  виртуальную визитку (если есть телефон).


- Включить в кампании **«Разметка ссылок для Метрики» (yclid)** — авто-связка
  с целями.
- `{keyword}`, `{ad_id}` — подставляются Директом автоматически.

## 8. Аналитика и цели (Метрика 109130279)

- Счётчик уже стоит, цели заведены: **`lead-kontur-market`,
  `lead-ts-piot-provider`** (отправка заявки), **`cpa-click`** (клик по
  CPA-баннеру), плюс engagement-цели.
- **До запуска:** прогнать воркфлоу «Метрика — синхронизация целей», чтобы
  `lead-*`/`cpa-click` реально создались в счётчике (иначе оптимизировать
  не по чему).
- В Директе выбрать **ключевую цель = `lead-kontur-market`** (и `lead-*`)
  для стратегии «Оплата за конверсии»/«Максимум конверсий».
- Связать Директ ↔ Метрику (доступ счётчика), включить автотегирование.
- Отчёты: Мастер отчётов в Метрике по utm_campaign/term → стоимость лида
  по группам.

## 9. 38-ФЗ / маркировка рекламы

- **Объявления Директа = реклама НАШЕГО сайта.** Яндекс.Директ —
  оператор рекламных данных, маркирует креативы в ЕРИР **автоматически**.
  Отдельный erid на объявление вручную ставить не нужно.
- **Офферы Контура на посадочной** маркируются отдельно (erid уже
  проставлен в `CpaCallout`/`KonturOrderWidget`). Связка корректна.
- Не использовать в объявлениях «лучший», «№1», гарантии результата —
  риск отклонения модерацией и ФАС.

## 11. Оптимизация (первые 2 недели)

- **Ежедневно:** отчёт «Поисковые запросы» → добавлять минус-слова.
- **Каждые 3–4 дня:** отключать ключи/объявления с CPC высоким и 0 конверсий.
- **Через 7 дней:** свести стоимость лида по группам (utm) → масштабировать
  прибыльные, резать убыточные.
- **Через 14 дней:** при ≥10 конверсиях — перевести на «Максимум конверсий»
  с целевой ценой лида; решить про запуск K2–K4.
- **A/B:** держать 2 объявления на группу, слабое менять.

> Главное правило: не «крутить ради трафика», а гнать к цели `lead-*` и
> резать всё, что не окупается по CPA.
