# Roadmap: витрина продуктов Контура (CPA-storefront)

План превращения портала из «информационка + одиночные CPA-баннеры»
в полноценную витрину набора продуктов Контура. Фаза 1 — только
заявки (лиды), без онлайн-оплаты. Онлайн-оплата вынесена в задел
(раздел «Отложено»).

Статус: план. Код по этому документу ещё не написан.

---

## 1. Модель: агрегатор поверх биллинга Контура

Сайт статический (Astro → GitHub Pages), своего бэкенда нет. Поэтому
портал не принимает платежи, не хранит лиды и не выступает продавцом.
Роль — лучшая в нише воронка под продукты Контура. Всё, что «продаёт»,
живёт на стороне Контура.

Два механизма монетизации (термины не путать):

| Механизм | Что происходит | Где деньги |
|---|---|---|
| **Лид-форма с тегом** | Клиент оставляет заявку, не уходя с сайта. Заявка падает в кабинет партнёра с кодом `f74746`. Контур закрывает сделку. | CPA за продажу по лиду |
| **Онлайн-оплата (отложено)** | Партнёр генерирует ссылку на оплату/счёт Контура. Оплата идёт на `kontur.ru`. | CPA за оплату |

Фаза 1 использует **только лид-формы**. Это снимает проблему недоверия:
портал не имитирует свой магазин, заявку обрабатывает федеральный бренд.

Подтверждённая механика партнёрской программы Контура:
- встраиваемые **виджеты формы заявки** (HTML/iframe с партнёрским кодом,
  заявки уходят с тегом) — ложатся на статику без бэкенда;
- вознаграждение **5–50 %** в зависимости от продукта;
- ссылки на онлайн-оплату/счёт (для будущей фазы).

Источники: [kontur.ru/partnership](https://kontur.ru/partnership),
[kontur.ru/partnership/online](https://kontur.ru/partnership/online),
[support.kontur.ru/referral](https://support.kontur.ru/referral/38994-registraciya_v_programme).

## 2. Что уже есть в репозитории

- `src/data/cpa-banners.ts` — фактически готовая **база продуктов**:
  13 живых офферов Контура (ОФД, Маркет, Эльба, Экстерн, Фокус, Диадок,
  КЭДО, МЧД, Банк+Эльба и др.) с тегом `f74746` и автоматической
  маркировкой по 38-ФЗ через `erid` из `ord-erids.json`.
- `src/components/CpaCallout.astro` — рендер одного оффера по `id`,
  маркировка рекламы (`rel="sponsored"`, плашка `erid`) автоматическая.
- `CATEGORY_DEFAULT_CPA` / `CATEGORY_DEFAULT_CPA2` — дефолтный оффер
  по категории статьи.
- Сценарии (`/scenario/*`) и интерактивы (`UsnNdsCalc`,
  `PenaltyCalculator`, квизы) — готовые точки встройки.

То есть данные под каталог уже описаны. Не хватает страниц-витрин,
компонента-обёртки для виджета заявки и блоков-бандлов.

## 3. Связь с существующими планами

Чтобы не плодить дубли — этот roadmap опирается на уже отобранные блоки
из `docs/blocks-roadmap.md`:

- **LAB1 #2 — Quiz «подбор сервиса»** (`/podbor/`) — верхняя часть
  воронки: подбирает продукт под бизнес, ведёт на страницу продукта.
- **LAB1 #12 — Comparison table (F-27)** (`/sravneniya/`) — sticky-таблица
  сравнения, конверсионный инструмент. Страница продукта линкуется в неё.
- **Inline vs popup** (`docs/blocks-roadmap.md`, раздел про лид-формы) —
  обоснование, что inline-форма конвертит кратно выше ухода по ссылке.

Витрина — это «середина» воронки между подбором/сравнением и заявкой.

## 4. Фаза 1 — только заявки

### 4.1 Компонент `KonturLeadForm.astro` (ядро фазы)
Обёртка над официальным виджетом заявки Контура. Принимает `productId`
(ключ из `CPA_BANNERS`), подставляет нужный виджет/iframe с тегом,
рисует триггеры доверия (аккредитация, «1,5 млн компаний», логотип),
маркировку по 38-ФЗ (`erid`). Встраивается в статьи (F26-блок),
спецстраницы и страницы продукта.

Критично: показ виджета рекламного оффера **без `erid`** — нарушение
38-ФЗ. Гейт на сборке: если у оффера нет `erid`, форма не рендерится
(как уже сделано для банковских заготовок в `cpa-banners.ts`).

### 4.2 Раздел-каталог `/produkty/`
- Индекс: карточки продуктов, сгруппированные по задачам бизнеса —
  касса и фискализация (Маркет, ОФД, ТС ПИоТ), бухгалтерия и отчётность
  (Эльба, Экстерн), ЭДО и кадры (Диадок, КЭДО, МЧД), проверки и риски
  (Фокус), деньги (Банк+Эльба).
- Страница продукта `/produkty/<id>/`: что это, кому, тариф/условия,
  `KonturLeadForm`, перелинковка в статьи кластера и в `/sravneniya/`.
- Источник данных — `CPA_BANNERS`. Расширить интерфейс `CpaBanner`
  опциональными полями для витрины (целевая аудитория, буллеты выгод,
  группа каталога), не ломая текущий рендер баннеров.

### 4.3 Бандлы под сценарий
Блок «набор продуктов под задачу» в существующих сценариях
(`/scenario/kofeynya-za-30-dney`, `perekhod-ip-na-ooo`): 3–4 связанных
оффера одним блоком (кофейня → касса+ТС ПИоТ + ОФД + Эльба + Диадок).
Поднимает средний чек, попадает в реальную потребность. Зачаток —
связка-оффер `bank-elba` в данных.

### 4.4 Перелинковка воронки
`/podbor/` (quiz) → страница продукта `/produkty/<id>/` → `/sravneniya/`
(если есть сравнение) → лид-форма. Из статей кластера — ссылки на
профильные страницы продуктов и бандлы.

## 5. Лечение недоверия (даже в фазе заявок)

- Не имитировать свой магазин. Формулировки честные: «оставить заявку —
  Контур перезвонит», «оформление на kontur.ru, договор с Контуром».
- Триггеры доверия рядом с формой: аккредитация оператора, число
  клиентов, логотип, видимая маркировка рекламы.
- Дорогие/сложные продукты (Маркировка под ключ, ЭДО-интеграция) —
  только заявка; они и так продаются через звонок менеджера.

## 6. Метрики и проверка гипотез

- Цели Метрики на отправку заявки по каждому виджету (см. `docs/metrika.md`,
  `src/data/metrika/goals.json`). Обёртка `src/utils/track.ts` —
  `reachGoal` на submit.
- Сравнить конверсию inline-формы против старого «ухода по ссылке»
  (`CpaCallout` с внешним `ctaHref`) на одном и том же оффере.
- Решение о фазе 2 (онлайн-оплата) — по данным фазы 1: если конверсия
  в заявку и доходимость до сделки устраивают, добавлять оплату точечно.

## 7. Отложено (не делаем сейчас)

- **Онлайн-оплата прямо на сайте** — кнопка с готовой ссылкой оплаты
  Контура. Подключать после замера фазы 1, начиная с дешёвых понятных
  продуктов (ОФД, Фокус). Платёж — на домене Контура, не у портала.
- **Прайс-калькулятор «собери тариф»** с выводом ссылки на оплату.
- **Реселлерская модель** (продажа от своего ИП, свои счета) —
  отклонено: другой бизнес (биллинг, поддержка, возвраты), убивает
  скорость и доверие. Реферальная модель даёт те же 5–50 % без этого.
- **Банковские партнёрки** (Т-Банк, Точка, Альфа) — заготовки в
  `cpa-banners.ts` без `erid` и без партнёрских ссылок. Не показывать,
  пока не одобрены заявки и не зарегистрированы креативы в ОРД.

## 8. Открытые вопросы

- Точный формат виджета Контура: iframe или вставка скрипта. Нужен
  HTML из кабинета партнёра по каждому продукту.
- Какие из 13 офферов имеют готовый виджет заявки, а какие — только
  реферальную ссылку (для них фаза 1 = `CpaCallout`, без inline-формы).
- erid: проверить покрытие в `ord-erids.json` для офферов, идущих в
  витрину (`scripts/ord-register.mjs`).
