# Журнал backfill фактчека

Закрытие задач [#39](https://github.com/burbonivanovich-oss/TS_PIOT/issues/39)
и #58 (локальный backlog) — прохождение всех 83 опубликованных статей
блога через `/factcheck`.

## Итог

| Дата | Покрытие |
|---|---|
| До сессии | 8/83 (10%) |
| 2026-05-19 (одна сессия) | **83/83 (100%)** |

С 19.05.2026 каждая опубликованная статья имеет маркер
`.claude/factchecked/<slug>` и JSON-отчёт `src/data/factcheck/results/<slug>.json`.

## Процесс

18 параллельных батчей через `general-purpose` subagent-ы, по 4–5 статей
каждый. Один батч ≈ 5 минут реального времени. Каждый агент работал по
протоколу:

1. `node scripts/factcheck/extract-claims.mjs <slug>` — извлечение claims.
2. WebSearch по каждому claim против `src/data/factcheck/sources.json`.
3. Классификация по `docs/editorial-policy.md` (A/B/C).
4. Точечная правка статьи при class C, `updatedDate: 2026-05-19`.
5. Запись `src/data/factcheck/results/<slug>.json`.
6. Маркер `.claude/factchecked/<slug>`.

Группировка батчей по риску:
* **Первая волна:** ТС ПИоТ + штрафы маркировки + налоговые сроки 2026
  (80% репутационного риска).
* **Вторая волна:** маркировка по категориям, ЕГАИС, Меркурий.
* **Третья волна:** ккт-обзоры моделей, ИП/найм/общепит — низкий
  нормативный риск.

## Системные ошибки за пределами фейков

Кроме «выдуманных номеров», выявлены повторяющиеся ошибки в фактологии:

### Налоговая реформа 2026 (повторение в 6+ статьях)
* НДС основная 20% → **22%** (ФЗ № 425-ФЗ от 28.11.2025)
* Порог НДС-на-УСН 60 млн → **20 млн**
* Лимит выручки УСН 450 млн → **490.5 млн** (с дефлятором 1.090, ПП Минэк № 734)
* Лимит выручки ПСН 60 млн → **20 млн** (синхрон с НДС)
* Пониженные ставки УСН — пороги 20–272.5 / 272.5–490.5 млн (не 60–250 / 250–450)
* НДФЛ пятиступенчатый 13/15/18/20/22% (не 13/15)
* МРОТ 2025 22 440 ₽ (не 25 371); МРОТ 2026 27 093 ₽
* Порог 1.5 МРОТ для пониженного тарифа МСП 15% (не «свыше МРОТ»)

### Штрафы (повторение в 10+ статьях)
* ст. 15.12 КоАП без указания части — обычно ч. 2
* Штраф ИП по ст. 15.12 — 5–10 тыс ₽ (не 5–15 — это устаревшая редакция до 2017)
* Штраф юрлиц по ст. 15.13 КоАП — 50–100K (не 200K)

### ЕГАИС / алкоголь
* «Крепкий алкоголь >15%» (на самом деле **>9%** по 171-ФЗ) — 2 независимых случая
* Форма 8 декларации отменена с 15.01.2025 (последняя сдача — апрель 2025)
* ФСРАР преобразован в **Росалкогольтабакконтроль** с 16.02.2024
* Маркировка кег действует с 01.04.2023, поэкземплярный учёт с 01.03.2025

### Маркировка
* Путаница «обязательная маркировка» / «разрешительный режим в рознице» / «вывод из оборота через кассу» — три разные даты, выдаются за одну
* Игрушки 01.09.2026 — не «старт маркировки», а конец перехода
* Гигиена ПП № 1457 vs № 1681 — добровольный эксперимент выдан за обязательную маркировку

### Маркетплейсы
* Дата 289-ФЗ 08.08.2024 → **31.07.2025** (название тоже было неверным)
* Критерий «крупных платформ» 30 млрд ₽ → **аудитория ≥ 100 000**
* Срок уведомления о смене условий 30 → 45 дней
* НПД на WB — 4% → **6%** (WB как юрлицо, ст. 10 422-ФЗ)

## Как это закрыто на пайплайн-уровне

Все пойманные паттерны теперь системно недопустимы для новых статей
через [#61 / GitHub #31](https://github.com/burbonivanovich-oss/TS_PIOT/issues/31):

1. **`/create-article` Стадия 1.** Для категорий с НПА
   (ts-piot, markirovka, zakonodatelstvo, kkt, egais) запуск
   `content-writer` без `research-specialist` запрещён.
2. **`research-specialist`** обязан физически верифицировать каждую норму
   через WebFetch/WebSearch до сохранения брифа. Frontmatter brief'а
   содержит `sources[].verified=true`.
3. **`content-writer`** имеет жёсткое правило: использовать только нормы
   из `brief.sources[]` и `sources.json.npaWhitelist`. Иначе плейсхолдер
   `[НПА: нужна проверка]`.
4. **`/create-article` Стадия 3.** Прогон
   `node scripts/factcheck/audit-npa-references.mjs --strict` до запуска
   `/factcheck` — упадёт на любом упоминании ПП/Приказа/ФЗ, которого нет
   в whitelist.
5. **`extract-claims.mjs`** теперь ловит NPA_UK / NPA_NK /
   NPA_PP_NUMBERED / NPA_PRIKAZ — `/factcheck` имеет шанс поймать
   подозрительные номера на стадии экстракции.

## Открытые риски

### Реальный номер не той нормы

Самый сложный к автоматическому отлову паттерн (1 случай за сессию):
ПП № 2099 от 15.12.2020 — реальное действующее постановление о
маркировке молочной продукции — было указано в статье
`kak-vybrat-onlayn-kassu-2026` как нормативка для ТС ПИоТ.
`audit-npa-references.mjs` сверяет только номера и подтвердил бы
существование № 2099, не заметив смены контекста.

Решение для будущей итерации: в `npaWhitelist.{pp,fz}.<num>.topic`
у нас уже есть категория нормы. Расширить `audit-npa-references.mjs`
семантической проверкой — если ПП цитируется в статье с категорией X,
а его `topic` = Y, помечать class B (требует ручной верификации).

### 37 ФЗ + 18 ПП вне whitelist

Первый прогон `audit-npa-references.mjs` показал 37 незнакомых ФЗ
и 18 незнакомых ПП. Все они **могут быть реальными** — whitelist пока
содержит только базовые нормы наших тематик. После сессии в whitelist
добавлены 7 верифицированных норм (ФЗ № 362/365/391/429, Приказы № 396/648,
Распоряжение № 4125-р), но картина по 50+ упоминаниям ещё неполная.

Кандидат на отдельный цикл: пройти оставшиеся незнакомые номера,
верифицировать через WebSearch, дополнить whitelist. После этого
`audit-npa-references.mjs --strict` начнёт давать осмысленный сигнал
для CI.
