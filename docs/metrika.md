# Яндекс.Метрика: цели через Management API

Цели Метрики у нас живут в репозитории и синхронизируются со счётчиком
автоматически. Это даёт code review, версионирование, и снимает
необходимость вручную кликать в UI Метрики при добавлении каждого
нового интерактива.

## Что где

| Файл | Назначение |
|---|---|
| `src/data/metrika/goals.json` | **Источник истины.** Декларативный список целей: id, name, тип, опциональный комментарий |
| `scripts/metrika/sync-goals.mjs` | Идемпотентный синхронизатор: создаёт недостающие, обновляет имена, не удаляет orphan-цели |
| `.github/workflows/metrika-sync-goals.yml` | Триггеры: ручной `workflow_dispatch` и автоматический на push в `goals.json` (только main) |
| `src/utils/track.ts` | Клиентская отправка `track('goal-id')` → `window.ym(109130279, 'reachGoal', ...)` |

Счётчик: **109130279** (подключён в `src/components/BaseHead.astro`).

## Как добавить новую цель

1. Открыть `src/data/metrika/goals.json`.
2. Добавить запись в массив `goals`:
   ```json
   {
     "id": "calc-new-tool-used",
     "name": "Новый калькулятор — выполнен расчёт",
     "type": "action",
     "comment": "опционально"
   }
   ```
3. В коде компонента вызывать `track('calc-new-tool-used')` или
   `trackOnce('calc-new-tool-used')` из `src/utils/track.ts`.
4. Закоммитить и запушить в main → workflow `Metrika — Sync Goals`
   автоматически создаст цель на счётчике.

**Правило:** не меняйте `id` после того, как цель попала в main.
Это идентификатор, привязанный к `conditions[0].url` в Метрике —
если поменять, скрипт создаст новую цель и оставит старую (с
накопленной аналитикой) как orphan. `name` менять можно — sync
обновит человеко-читаемое имя в UI.

## Первичная настройка (один раз)

### 1. Получить OAuth-токен

1. Перейти на [oauth.yandex.ru](https://oauth.yandex.ru/) под аккаунтом,
   у которого есть доступ к счётчику 109130279 как «Представитель —
   редактирование».
2. **Зарегистрировать приложение** (если ещё не сделано):
   - Платформа: «Веб-сервисы».
   - Redirect URI: `https://oauth.yandex.ru/verification_code`
     (для упрощённого получения токена).
   - **Доступы:** выбрать «Яндекс.Метрика → Управление счётчиками,
     включая параметры обработки и доступа» (`metrika:write`).
3. После создания приложения нажать «Получить токен» —
   Яндекс выдаст строку вида `y0_AgAAA...`.

### 2. Прописать секрет в GitHub

GitHub → репозиторий `burbonivanovich-oss/TS_PIOT` → Settings →
Secrets and variables → Actions → New repository secret:

- Name: `METRIKA_OAUTH_TOKEN`
- Secret: `y0_AgAAA...` (значение токена).

### 3. Первый прогон в DRY_RUN

Actions → `Metrika — Sync Goals` → Run workflow → оставить
`dry_run: 1`.

Скрипт выведет план: сколько целей будет создано, сколько
обновлено, какие orphan. Если план выглядит здраво —

### 4. Запустить боевой прогон

Тот же workflow с `dry_run: 0`. Создаст недостающие цели.
Дальше работает автоматически: push в `src/data/metrika/goals.json` →
sync.

## Безопасность

- Скрипт **никогда не удаляет** цели — только создаёт и
  обновляет имена. Лишние orphan-цели остаются, чтобы не
  потерять накопленную аналитику. Удалять — вручную в UI
  Метрики после ревью.
- DRY_RUN-режим (default для `workflow_dispatch`) пишет план
  и выходит, не дёргая POST/PUT.
- Токен живёт только в секретах GitHub Actions, в репо его нет.

## Открытые вопросы

1. **URL-цели и составные.** Сейчас поддерживаются только
   action-цели (`window.ym('reachGoal', '...')`). Если потребуется
   URL-цель («посетил `/kak-rabotaet-ts-piot/`») — расширить
   `asActionPayload()` и схему JSON. Не делаем пока без
   реальной потребности.
2. **Цели-воронки.** В Метрике есть составные цели «по шагам»
   (type `step`). 9 шагов флагмана — естественный кандидат.
   Сейчас храним их как 9 отдельных action-целей и собираем
   воронку через отчёт «Конверсии». Если окажется нужно — добавим
   тип `step` в схему отдельно.
3. **Удаление orphan по флагу.** Опасно, но иногда нужно
   (например, после рефакторинга имён). Можно добавить флаг
   `DELETE_ORPHANS=1`, но только после двойного подтверждения и
   с предварительным DRY_RUN. Не делаем до явной потребности.
