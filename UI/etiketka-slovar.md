# Словарь / Глоссарий (etiketka-media.ru/slovar) — CPA-аудит

## Оценка: 4/10

## CPA-потенциал
- **Тип трафика:** холодный и тёплый — SEO-трафик по брендовым запросам аббревиатур («ТС ПИоТ что это», «ОФД расшифровка», «ЭДО что такое для бизнеса»). Пользователь пришёл разобраться в терминах — он на стадии изучения проблемы.
- **Целевое действие:** дочитать определение → перейти по ссылке «Категория» или «Статья» → попасть на страницу с партнёрским размещением. Прямой конверсии с глоссария нет и быть сложно, но это страница-прогрев.
- **Текущий CPA-статус:** партнёрских ссылок нет. Есть единственная внутренняя ссылка «Калькулятор штрафов →» у терминов ЕГАИС, Меркурий, МРЦ — это правильный шаг, но не CPA-монетизация.

---

## Что работает

1. **Богатая структура глоссария.** 35+ терминов, алфавитная навигация, aliases (Также: ...), JSON-LD DefinedTermSet — хорошая SEO-база для захвата трафика по аббревиатурным запросам.
2. **Категориальные метки** у каждого термина (таблетка «Маркировка», «ТС ПИоТ», «Законодательство») работают как внутренние ссылки на разделы, создавая навигационный поток в контент.
3. **Перекрёстные ссылки между терминами.** Например, у «Вывод из оборота» → «Код маркировки», у «ККТ» → «ТС ПИоТ». Это удерживает пользователя на сайте.
4. **Ссылки на Калькулятор штрафов** у терминов ЕГАИС, Меркурий, МРЦ — единственные элементы, ведущие на инструментальную страницу с потенциалом монетизации.
5. **Техническая скорость отличная.** LCP 192 мс — страница открывается мгновенно, несмотря на объём (109 КБ HTML из-за JSON-LD).

## CPA-проблемы (приоритизированные)

### #1 — критично: Термины ОФД, ЭДО, ККТ не содержат партнёрских CTA

**Описание:** Термин «ОФД» — оператор фискальных данных — это прямой покупательский интент: предприниматель, ищущий расшифровку, скорее всего выбирает или только подключает ОФД. В карточке термина есть только ссылки «Что такое ТС ПИоТ →» и «Категория: ТС ПИоТ →». Аналогично с «ЭДО» (нет партнёра), «ККТ» (нет ссылки на партнёра по кассам), «УКЭП» (нет ссылки на удостоверяющий центр).

**Рекомендация:** В карточки терминов «ОФД», «ЭДО», «ККТ», «УКЭП», «Смарт-терминал» добавить блок «Рекомендуем» с партнёрской ссылкой после определения. Формат: небольшая серая плашка «Ищете [термин]? Рекомендуем: [Партнёр] — [ссылка]». Это органично вписывается в образовательный формат страницы.

**Усилие:** M (шаблон плашки + интеграция в Astro-компонент термина)

---

### #2 — критично: Отсутствует раздел «Полезные сервисы» или сайдбар

**Описание:** Страница глоссария — длинный одноколоночный список без боковой панели. В B2B-медиа с регуляторной тематикой принято иметь sticky-сайдбар с 2–3 партнёрскими блоками («Нужен ОФД?», «Подключить ЭДО»), которые остаются видимыми при прокрутке длинного контента.

**Рекомендация:** Переверстать страницу глоссария в двухколоночный лейаут на desktop: основная колонка (820px, текущая) + правый сайдбар (280px) со sticky-позиционированием. В сайдбаре — 2–3 партнёрских карточки, обновляемые по сезону (сейчас актуально: ОФД для ТС ПИоТ, ЭДО для маркировки).

**Усилие:** L (верстка нового layout)

---

### #3 — высокий: Термины с высоким intent не ведут на конвертирующие страницы

**Описание:** «Разрешительный режим», «ТС ПИоТ», «ФН (фискальный накопитель)» — термины, которые ищут предприниматели с активной проблемой (дедлайн уже наступил). В карточке «ТС ПИоТ» есть ссылка «Что такое ТС ПИоТ →» и «Категория: ТС ПИоТ →». Но нет прямой ссылки на материал «ОФД для ТС ПИоТ 2026» — ближайшую статью к конверсии.

**Рекомендация:** В карточке «ТС ПИоТ» добавить ссылку: «Как выбрать ОФД для ТС ПИоТ →» (ведёт на статью `/blog/2026-05-01-ofd-dlya-ts-piot/`). Аналогично для «ОФД» — добавить «Сравнение операторов ОФД →» или прямую партнёрскую ссылку.

**Усилие:** S (добавить ссылки в JSON-данные терминов)

---

### #4 — высокий: Нет CTA после алфавитного навигатора

**Описание:** Пользователь, пришедший на страницу глоссария, видит алфавитный навигатор (В, Е, К, Л...) и сразу попадает в список. Выше алфавитного навигатора нет никакого призыва к действию — даже нет анонса «Если вам нужна помощь с [термином], обратитесь к [партнёр]».

**Рекомендация:** Добавить небольшой инфо-блок между описанием страницы и алфавитным навигатором: «Работаете с маркировкой? Проверьте, готова ли ваша касса: [ссылка на партнёра или на Калькулятор штрафов]».

**Усилие:** S

---

### #5 — средний: Нет ссылки на Калькулятор штрафов в карточке «ФНС», «НДС», «УСН»

**Описание:** Термины НДС, УСН, ОСНО — налоговая тематика. После прочтения определения у пользователя нет явного следующего шага. Калькулятор штрафов при этом охватывает налоговые нарушения (ЕНС, ненаправление уведомлений), что релевантно.

**Рекомендация:** В карточках НДС, УСН, ЕНС/ЕНП добавить ссылку «Калькулятор штрафов по налоговым нарушениям →» — это и внутренняя перелинковка для SEO, и шаг к монетизируемой странице.

**Усилие:** S

## Быстрые CPA-победы (реализация менее 1 дня)

1. **В карточку «ОФД»** добавить строку «Найти и подключить ОФД: [Партнёр 1], [Партнёр 2]» с UTM-ссылками — это один из самых горячих терминов страницы. Буквально правка в data-файле Astro.
2. **В карточку «ЭДО»** добавить «Попробовать ЭДО бесплатно: [Партнёр] →» — маркировка товаров требует обязательного ЭДО, аудитория имеет прямой intent.
3. **В карточку «ТС ПИоТ»** добавить ссылку на `/blog/2026-05-01-ofd-dlya-ts-piot/` с текстом «Как выбрать ОФД для ТС ПИоТ →» — статья уже содержит (или может содержать) партнёрский блок, глоссарий станет воронкой на неё.
