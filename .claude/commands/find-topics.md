# /find-topics — найти темы для статей

Анализирует что уже опубликовано, сверяет с контент-планом, ищет незакрытые
запросы через поиск. Возвращает приоритизированный список тем готовых к
написанию.

## Алгоритм

### Шаг 1. Аудит текущего контента

1. Прочитать `src/content/wiki/content-plan-2026.md` — кластеры, слаги, статусы.
2. Прочитать все файлы `src/content/blog/*.md` — извлечь slug, title, categories,
   tags, pubDate. Собрать список того, что уже опубликовано (draft: false).
3. Сопоставить с контент-планом: какие строки помечены P0/P1, но ещё не
   опубликованы.

### Шаг 2. Поиск новых запросов

Для каждого активного кластера (или указанного в аргументе) выполнить WebSearch:

- `"[тема кластера] 2026" site:sudact.ru OR site:nalog.ru OR site:честныйзнак.рф`
  — свежие нормативные изменения.
- `"[тема кластера] как" OR "[тема кластера] что такое" OR "[тема кластера] штраф"`
  — информационные запросы с высоким интентом.
- Поискать вопросы из «Люди также спрашивают» Яндекса по ключевым темам.
- Проверить, не появились ли новые сроки или изменения законодательства
  после даты последней публикации в кластере.

### Шаг 3. Проверка на каннибализацию

Для каждой найденной темы проверить — нет ли уже похожей статьи в блоге
(пересечение ключей > 50%). Если есть — предложить обновление, а не новую
статью.

### Шаг 4. Приоритизация

Ранжировать темы по:
1. **Срочность** — есть конкретный дедлайн / вступающий в силу закон.
2. **Поисковый спрос** — судя по частоте в результатах поиска и вариативности
   запросов.
3. **Пробел в кластере** — нет покрытия ни одной статьёй.
4. **Сложность написания** — простые информационные статьи в приоритете над
   сложными аналитическими.

## Формат вывода

Таблица Markdown, готовая к вставке в `content-plan-2026.md`:

```markdown
| # | Слаг | Тема | Целевой запрос | Приоритет | Обоснование |
|---|------|------|---------------|-----------|-------------|
| 1 | slug-temы | Заголовок темы | ключевой запрос | P0 | Срок вступления в силу: ДД.ММ.ГГГГ |
```

Плюс отдельный блок «Рекомендую обновить» — статьи старше 6 месяцев по теме,
у которых появились новые НПА или изменились сроки.

## Ограничения

- Не предлагать темы, которые дублируют существующие статьи.
- Не предлагать коммерческие темы («лучший сервис», «топ-10 касс») — только
  информационный интент.
- Минимум 5, максимум 15 новых тем за один запуск.
