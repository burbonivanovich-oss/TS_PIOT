# /update-template — обновить брендинг

Используется, когда нужно изменить ключевые «глобальные» поля сайта без
переписывания кода руками. Команда проходится по местам, где зашит брендинг.

## Что обновляется

| Файл | Что менять |
| --- | --- |
| `src/consts.ts` | `SITE_TITLE`, `SITE_DESCRIPTION`, `SITE_URL`, `SITE_AUTHOR`, `NAV_LINKS`, `CATEGORIES` |
| `astro.config.mjs` | `site` (URL продакшена) |
| `CLAUDE.md` | Описание тематики и правил контента |
| `package.json` | `name` (kebab-case) |
| `src/pages/about.astro` | Описание проекта |
| `src/pages/privacy.astro` | Реквизиты оператора и контакты |
| `public/robots.txt` | Sitemap-URL и Host |
| `public/og-default.svg` | Текст в OG-плейсхолдере |
| `src/components/Footer.astro` | Дисклеймер, контакты |

## Что не меняется автоматически

- Содержимое статей (`src/content/blog/`).
- Структура агентов и команд.
- Стили (`src/styles/global.css`) — изменяются отдельным запросом.

## Параметры запуска

При запуске команды Claude задаст вопросы:

1. **Новое название бренда** (например, «Закон.Контроль»).
2. **Новый домен** (например, `https://zakon-kontrol.ru`).
3. **Новый набор категорий** (если меняется фокус) — список slug + русское
   название + описание.
4. **Контактный email** для дисклеймера и privacy.

После ответов Claude в одном проходе обновляет все файлы из таблицы выше и
запускает `npm run build`, чтобы убедиться, что ничего не сломалось.
