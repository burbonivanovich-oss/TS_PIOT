# /check-ai — детекция AI-текста

Анализирует статью на признаки нейросетевого происхождения. Не блокирует
публикацию — даёт конкретные правки для «очеловечивания» текста.

## Что проверяется

### 1. Стоп-фразы (явные AI-маркеры для русского текста)

Канцеляризмы, которые нейросети воспроизводят чаще людей:

**Вводные и связки:**
- «следует отметить», «следует также отметить»
- «важно понимать», «важно отметить», «важно учитывать»
- «необходимо учитывать», «необходимо отметить»
- «в заключение можно сказать», «подводя итог»
- «таким образом» (в начале абзаца)
- «вместе с тем», «помимо этого»
- «в данном контексте», «в рамках данной статьи»
- «одним из ключевых», «ключевым аспектом является»

**Пассивные глагольные конструкции:**
- «является» (вместо «—» или живого глагола)
- «осуществляется», «осуществлять»
- «производится», «производить» в значении «делать»
- «реализуется», «реализовывать» в значении «выполнять»
- «применяется», «применяться»

**Канцелярные существительные:**
- «данный», «данная», «данное» (вместо «этот», «эта»)
- «актуальным является», «немаловажным является»
- «в настоящее время» (вместо «сейчас»)
- «на сегодняшний день»
- «надлежащим образом»
- «в полном объёме»
- «конечный потребитель» (вместо «покупатель»)

**Оценочные клише без конкретики:**
- «играет важную роль»
- «оказывает существенное влияние»
- «в целом можно сказать»
- «в целом», «как правило» (чрезмерно)

### 2. Структурные паттерны AI

- Все абзацы примерно одинаковой длины (3–4 предложения каждый).
- Каждая H2-секция заканчивается обобщающим абзацем в духе «таким образом...».
- Списки везде ровно по 3 или 5 пунктов.
- Нет конкретных примеров из практики — только обобщения.
- Нет прямой речи, диалогов, цитат реальных людей.
- Нет разрывов ритма: текст читается равномерно без акцентов.

### 3. Специфика для B2B-тематики

- Нормы и статьи КоАП упомянуты без конкретных цифр.
- Сроки названы расплывчато («в ближайшее время», «в скором времени»).
- Отсутствуют исключения и граничные случаи — AI часто их опускает.

## Формат вывода

```
## AI-детекция: <название файла>

**Риск-скор:** X/10  (0 = человеческий текст, 10 = явный AI)

### Найденные стоп-фразы (N штук)
| Строка | Фраза | Замена |
|--------|-------|--------|
| 42 | «следует отметить, что» | удалить или «обратите внимание:» |
| 87 | «данный порядок является» | «этот порядок —» |

### Структурные признаки
- [ ] Длина абзацев однородна: да/нет
- [ ] Все секции с обобщающим выводом: да/нет
- [ ] Конкретные примеры присутствуют: да/нет

### Рекомендации
1. Заменить N стоп-фраз (список выше).
2. [конкретные советы по структуре]
3. Добавить пример из практики в секцию «[название]».
```

**Скор-скала:**
- 0–2 — публиковать без правок.
- 3–5 — заменить стоп-фразы, проверить структуру.
- 6–8 — переписать проблемные секции.
- 9–10 — переписать статью целиком с нуля от живого голоса.

## Запуск автоматической проверки

Параллельно запустить скрипт для машинного подсчёта маркеров:
```bash
node scripts/check-ai-markers.mjs <путь к файлу>
```

Скрипт выдаёт число вхождений каждой стоп-фразы — используй как ориентир,
а не как окончательный вердикт.
