# /demand-watch — watch спроса по кластерам

Сверяет опубликованные статьи с актуальной динамикой Wordstat (`keys.json`) и
выдаёт конкретные правки: какие статьи **обновить** (растущий спрос), где
**переформулировать** целевой ключ (аудитория ищет короче), что **написать**
(непокрытый спрос) и что **объединить/архивировать** (падающий спрос).

Это не поиск новых тем (`/cluster-gaps`) и не полный аудит статьи
(`/maintain-content`), а быстрый разбор «спрос ↔ покрытие» поверх свежих цифр.

Вызов без аргумента — все кластеры. С аргументом — один: `/demand-watch ts-piot`.

Кластеры берутся из `frontmatter.categories`: `ts-piot`, `markirovka`,
`zakonodatelstvo`, `kkt`, `egais` (и любые новые, если появятся).

---

## Источник данных

Отчёт готовит детерминированный скрипт
`scripts/wordstat/demand-watch.mjs` → `src/data/wordstat/demand-watch.md`.
Он запускается автоматически в конце weekly-workflow (после `fetch`/`discover`/
`diff`), поэтому к понедельнику отчёт уже свежий.

### Шаг 0 — Убедиться, что отчёт актуален

1. Прочитать `src/data/wordstat/demand-watch.md`. В шапке есть дата генерации и
   `lastFullUpdate` из `keys.json`.
2. Если файла нет, он старше последнего `lastFullUpdate` в `keys.json`, или
   старше 7 дней — перегенерировать локально (сетевых запросов нет, читает кеш):

   ```bash
   node scripts/wordstat/demand-watch.mjs            # все кластеры
   ONLY_CLUSTER=ts-piot node scripts/wordstat/demand-watch.mjs   # один
   ```

3. Для детализации читать `src/data/wordstat/keys.json` напрямую
   (`shows`, `topShows`, `topPhrase`, `trend`, `history`, `related`).

## Как читать буклеты отчёта

Скрипт уже разложил сигналы по четырём корзинам на каждый кластер. Задача
скилла — превратить их в конкретные действия.

### 📈 Растёт — обновить и расширить

Спрос на целевой ключ вырос ×2 (`recent3/baseline6`) или `trend = up`.

- Открыть статью, проверить, отвечает ли она на текущий интент.
- Предложить: расширить раздел под ключ, добавить FAQ-вопрос дословной
  формулировкой, обновить `updatedDate`.
- Высокий спрос (≥ 2000) + рост → приоритет P0.

### 🔁 Переформулировать — topShows ≫ shows

`topShows ≥ 3 × shows`: целевой ключ в `seo.keywords` сформулирован уже, чем
ищет аудитория. Скрипт подсказывает `topPhrase`.

- Предложить заменить/добавить ключ на `topPhrase` в `seo.keywords`,
  при необходимости — в title/description/H2.
- Не выкидывать старый ключ молча — добавляем более частотную формулировку.

### 🆕 Непокрытый спрос — новый материал/раздел

Фраза с якорным словом кластера и спросом ≥ 800, которой нет ни в одном
`seo.keywords` кластера.

- **Важно:** это эвристика по якорным словам, возможен межкластерный шум.
  Сначала проверить интент фразы (при сомнении — `topRequests`/`related` в
  `keys.json`).
- Если интент реальный и пробел подтверждён:
  - близкая тема к существующей статье → предложить **раздел** в ней;
  - самостоятельный интент с большим спросом → предложить **новую статью**
    через `/create-article` или занести в `content-plan-2026.md` через
    `/cluster-gaps`.

### 📉 Падает — объединение/архив

Спрос упал ×2. Кандидат на объединение с более крупной статьёй (canonical) или
архивацию (`draft: true`). Решение — через `/maintain-content` (там вся
логика merge/archive).

## Шаг 1 — Сводка и план

Вывести по запрошенным кластерам компактный план:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 /demand-watch — YYYY-MM-DD (данные Wordstat от <lastFullUpdate>)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## Кластер: ts-piot

ОБНОВИТЬ (растущий спрос):
  [slug] — ключ "<...>" вырос ×N (спрос <demand>) → [что добавить]

ПЕРЕФОРМУЛИРОВАТЬ:
  [slug] — "<ключ>" (shows N) → "<topPhrase>" (topShows M)

НАПИСАТЬ (непокрытый спрос):
  "<фраза>" — спрос M → раздел в [slug] / новая статья

ОБЪЕДИНИТЬ/АРХИВ:
  [slug] — спрос упал ×N → передать в /maintain-content
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Спросить подтверждение перед любыми правками.

## Шаг 2 — Выполнение (после подтверждения)

| Корзина | Действие |
|---|---|
| 📈 Растёт | Точечные правки в статье + `updatedDate` + `reviewDate` +3 мес. После правки фактов — `/factcheck <slug>`. |
| 🔁 Переформулировать | Обновить `seo.keywords` (+ при необходимости title/description/H2) на `topPhrase`. |
| 🆕 Непокрытый | Раздел в статье **или** `/create-article "<тема>"` — по согласованию. |
| 📉 Падает | Передать slug в `/maintain-content` для решения merge/archive. |

Правила правок (из `docs/content-rules.md`): активный залог, без стоп-слов
(`является`, `каковы`, `таким образом`, `молочка` и т. п.), числа цифрами,
даты конкретно, ≥ 3 внутренних ссылок. Менять только то, что в плане, —
не рефакторить стиль всей статьи.

Коммит на каждую статью: `update(content): <slug> — спрос ↑/переформулировка`.
После правок собрать список URL обновлённых статей для переобхода (Вебмастер /
GSC), как в `/maintain-content`.

---

## Правила

- Ничего не править без подтверждения пользователя.
- Не выкидывать рабочие ключи — только добавлять более частотные.
- 🆕 «Непокрытый спрос» проверять на интент перед написанием — там бывает шум.
- Крупные решения (переписать с нуля, объединить, архивировать) — отдавать в
  `/maintain-content` / `/create-article`, не дублировать их логику здесь.
- Язык отчёта и правок — только русский.
