# /cluster-gaps — анализ пробелов кластера

Сравнивает опубликованные статьи и контент-план с типичными поисковыми запросами
аудитории МСБ. Находит пустые ниши и предлагает темы для заполнения.

Вызов без аргументов — анализирует все пять кластеров.
С аргументом — только указанный: `/cluster-gaps ts-piot`

Доступные кластеры: `ts-piot`, `markirovka`, `kkt`, `egais`, `zakonodatelstvo`

---

## Алгоритм

### Шаг 1 — Инвентаризация

Прочитать:
- frontmatter всех файлов `src/content/blog/*.md` (title, seo.keywords, categories, draft)
- `src/content/wiki/content-plan-2026.md`

Построить по каждому кластеру список:
- опубликованных тем (draft: false)
- запланированных тем (из контент-плана)
- ключевых запросов, которые уже охвачены

### Шаг 2 — Типовые запросы аудитории

Для каждого кластера проверить охват по эталонным группам запросов:

**ts-piot:**
- что такое ТС ПИоТ / как работает
- кому нужен / кого касается
- как подключить / установить
- модели касс (Эвотор, Атол, Штрих, Дримкас, MSPOS)
- штрафы за отсутствие
- ОФД и ТС ПИоТ
- переходный период / сроки
- ошибки и проблемы подключения

**markirovka:**
- какие товары подлежат маркировке
- как зарегистрироваться в «Честном знаке»
- маркировка по категориям (молочка, пиво, одежда, обувь, табак, БАД, вода и др.)
- разрешительный режим на кассе
- штрафы за нарушения
- маркировка остатков
- ЭДО и УПД при маркировке
- пилотные проекты

**kkt:**
- как выбрать кассу / модели
- регистрация в ФНС
- замена фискального накопителя
- выбор ОФД
- ФФД 1.2 / ФН-1.2
- ошибки на кассе
- касса для ИП без работников
- касса для онлайн-торговли

**egais:**
- как подключиться к ЕГАИС
- УТМ: установка и настройка
- журнал учёта алкоголя
- декларации по формам 7 и 8
- гашение ВСД в Меркурии
- пиво и слабоалкогольные напитки
- ЕГАИС для общепита
- штрафы за нарушения ЕГАИС

**zakonodatelstvo:**
- УСН 2026: лимиты, ставки, НДС
- ЕНС / ЕНП: уведомления, сроки
- МРОТ 2026
- страховые взносы МСП
- КЭДО: кому обязателен
- проверки бизнеса: мораторий 2026
- маркетплейсы: налоги и требования
- самозанятые: лимиты и статус

### Шаг 3 — Определение пробелов

Для каждой группы запросов:
1. Есть ли опубликованная статья, которая её закрывает?
2. Есть ли статья в контент-плане?
3. Если нет ни того, ни другого → **пробел**.

Оценить приоритет пробела:
- **P0** — тема напрямую связана с ближайшим дедлайном (≤ 60 дней) или крупным штрафом
- **P1** — высокочастотный запрос без конкурентной статьи на сайте
- **P2** — полезная тема, но не срочная

### Шаг 5 — Сохранение (опционально)

Если пользователь подтверждает:
1. Добавить P0 и P1 пробелы в `src/content/wiki/content-plan-2026.md`
   в соответствующий раздел кластера с пометкой `# [cluster-gaps YYYY-MM-DD]`.
2. Закоммитить с сообщением `Контент-план: пробелы кластера [название]`.

## Правила

- Не добавлять в план без подтверждения.
- Ограничить P0 — не более 3 на кластер, иначе теряется фокус.
- Язык отчёта — только русский.
