# /cluster-gaps — анализ пробелов кластера

Сравнивает опубликованные статьи и контент-план с типичными поисковыми запросами
аудитории МСБ. Находит пустые ниши и предлагает темы для заполнения.

Вызов без аргументов — анализирует все 18 кластеров.
С аргументом — только указанный: `/cluster-gaps ts-piot`

Доступные кластеры:

|---|---|---|

| `ofd-fn` | ОФД и фискальные накопители | Кластер 4 |
| `merkuriy` | Меркурий ВетИС | Кластер 5 |
| `kkt` | Выбор кассы, модели, регистрация | Кластер 6 |

## Алгоритм

### Шаг 1 — Инвентаризация

Прочитать:
- frontmatter всех файлов `src/content/blog/*.md` (title, seo.keywords, categories, draft)
- `src/content/wiki/content-plan-2026.md`

Построить по каждому кластеру список:
- опубликованных тем (draft: false)
- запланированных тем (из контент-плана)
- ключевых запросов, которые уже охвачены

### Шаг 3 — Определение пробелов

Для каждой группы запросов:
1. Есть ли опубликованная статья, которая её закрывает?
2. Есть ли статья в контент-плане?
3. Если нет ни того, ни другого → **пробел**.

Оценить приоритет пробела. Для каждого эталонного запроса смотрим
`src/data/wordstat/keys.json` и считаем «реальный спрос» =
`max(shows, topShows)`:

- **P0** — тема напрямую связана с ближайшим дедлайном (≤ 60 дней) или крупным
  штрафом, **или** реальный спрос ≥ 2000, **или** `trend = up`.
- **P1** — реальный спрос 500–2000 или `trend = stable` без конкурентной
  статьи на сайте.
- **P2** — реальный спрос < 500, полезная тема, но не срочная.

Если запроса нет в `keys.json` — считаем «частота неизвестна», ставим
ориентировочный P1 и помечаем `[needs-wordstat]`, чтобы при следующем
weekly-обновлении проверить.

Если `topShows >> shows` (в 3+ раза), формулировка эталонного запроса
слишком узкая — заменить её на `topPhrase` и пересчитать пробел.

## Правила

- Не добавлять в план без подтверждения.
- Ограничить P0 — не более 3 на кластер, иначе теряется фокус.
- Язык отчёта — только русский.
- Если в `content-plan-2026.md` появился новый кластер, эталонные запросы выше
  тоже расширить — иначе охват пропадёт из радара.
