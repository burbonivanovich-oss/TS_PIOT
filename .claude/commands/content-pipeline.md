# /content-pipeline — пакетный пайплайн

Запускает несколько прогонов `/new-post` подряд по темам из контент-плана
(`src/content/wiki/content-plan-2026.md`).

## Объёмы

- `day` — 1 статья (опорная или сателлит).
- `week` — 2 опорных + 4 сателлита.
- `month` — 4 опорных + 12–14 сателлитов.

## Тематический фокус (опционально)

- `ts-piot` — только материалы кластера ТС ПИоТ.
- `markirovka` — только маркировка.
- `zakonodatelstvo` — только законодательство.
- без фокуса — балансируем по контент-плану.

## Pipeline

1. Получить из контент-плана список тем нужного приоритета (P0 → P1 → P2),
   статус `pending`.
2. Для каждой темы:
   - `/research-topic "<тема>" standard`
   - `/new-post "<тема>" "<целевой запрос>"`
   - `/optimize-seo post standard <slug>`
3. После прогона:
   - Обновить статус в контент-плане (✅ опубликовано или draft).
   - Сформировать сводный отчёт по партии материалов.

## Защита от перегрузки

- Не более 6 материалов за один прогон в режиме `month`.
- Все драфты сохраняются с `draft: true` — публикация только после ручной
  верификации фактов.
- Каждый фрагмент пайплайна сохраняется отдельным коммитом в feature-ветке.

## Отчёт

```markdown
# Партия: <дата>, фокус: <категория>

## Готово
- [статья 1] — [URL] — статус: draft
- [статья 2] — [URL] — статус: draft

## Не удалось завершить
- [тема X] — причина: [нет первоисточников]

## Рекомендации по следующей партии
- ...
```
