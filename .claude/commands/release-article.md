# /release-article — публикация статьи

**Процесс 3 из 4.** Запускается после `/create-article`, когда статья прошла
все проверки и находится в `draft: true`.

Вызов: `/release-article 2026-05-01-ts-piot-shtrafy`

---

## Шаг 1 — Проверка готовности

Прочитать `src/content/blog/SLUG.md`. Убедиться:

```
[ ] draft: true (файл существует)
[ ] heroImage заполнен (если нет — предупредить, но не блокировать)
[ ] Нет пометок [нужна проверка] в тексте
[ ] reviewDate заполнен
[ ] pubDate совпадает с сегодняшней датой (скорректировать если нет)
```

Если файл не найден или `draft: false` — СТОП.

---

## Шаг 2 — Перелинковка

Найти 2–3 опубликованные статьи (`draft: false`), из которых уместно
сослаться на новую статью.

Алгоритм:
1. Прочитать frontmatter всех опубликованных статей.
2. Найти статьи с пересечением тегов или категорий с новой.
3. Для каждой кандидатуры прочитать текст и найти абзац, где упоминается
   тема новой статьи — туда вставить ссылку.
4. Формат: `[анкорный текст](/blog/SLUG/)`.

Вывести план перед вставкой:
```
Перелинковка:
  src/content/blog/EXISTING-1.md  → абзац "..." → добавить ссылку
  src/content/blog/EXISTING-2.md  → абзац "..." → добавить ссылку
```

Спросить подтверждение → внести правки.

---

## Шаг 3 — Публикация

```
1. Установить draft: false в frontmatter
2. Убедиться: pubDate = сегодня (YYYY-MM-DD)
3. git add src/content/blog/SLUG.md [изменённые файлы перелинковки]
4. git commit -m "pub: [title]"
5. git push
```

GitHub Actions автоматически запустит деплой.

**Шлюз:** push прошёл без ошибок, `draft: false` в файле.

---

## Шаг 4 — Соцсети

Проверить наличие социальных черновиков:
- `src/content/wiki/social/YYYY-MM-DD-slug.md` или
- `src/content/social/YYYY-MM-DD-slug-social.md`

Если черновики есть → показать их пользователю, спросить:
```
Черновики для соцсетей готовы. Что делать?
  1. Показать черновики для ручной публикации
  2. Создать новые черновики (/blog repurpose)
  3. Пропустить
```

Если черновиков нет → запустить агент `social-media-manager` с опубликованной статьёй.
Сохранить: `src/content/wiki/social/YYYY-MM-DD-slug.md` со статусом `status: ready`.

## Шаг 5 — Индексация

Вывести инструкцию для ручной индексации:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
Индексация (выполнить вручную)
━━━━━━━━━━━━━━━━━━━━━━━━━━
URL статьи: https://etiketka.media/blog/SLUG/

Яндекс.Вебмастер:
  webmaster.yandex.ru → Индексирование → Переобход страниц
  Вставить URL → Отправить

Google Search Console:
  search.google.com/search-console → Проверка URL
  Вставить URL → Запросить индексацию
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Правила

- Не публиковать статью без подтверждения перелинковки.
- Не устанавливать `draft: false` до успешного push.
- Язык — только русский.
