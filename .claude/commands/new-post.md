# /new-post — создать статью

Запускает полный пайплайн подготовки материала для портала. Аргументы:
- `"<тема>"` — обязательно, формулировка темы или рабочий заголовок;
- `[целевой запрос]` — опционально, основной поисковый ключ.

## Этапы

1. **Research** (агент `research-specialist`).
   - Собрать НПА, сроки, штрафы, исключения.
   - Сформировать research brief по шаблону агента.
   - Сохранить как `.claude/research/YYYY-MM-DD-slug-brief.md` (gitignore).

2. **Каннибализация** — до написания статьи.
   - Запустить `/blog cannibalization src/content/blog/`.
   - Если найдена статья с > 60% пересечением ключей — остановиться и предложить
     обновить существующий материал вместо создания нового.
   - Зафиксировать результат в research brief.

3. **Content** (агент `content-writer`).
   - На основе brief написать статью 1500–2500 слов на русском.
   - Структура: лид → 5–7 H2 → таблица/чек-лист → FAQ → заключение.
   - 3+ внутренних ссылок, ссылки на НПА.
   - Сохранить как `src/content/blog/YYYY-MM-DD-slug.md` с `draft: true`.

4. **SEO** (агент `seo-optimizer`).
   - Прогнать чек-лист (title, description, H1/H2, перелинковка, slug).
   - Предложить дополнительную разметку (FAQPage, HowTo) — если применимо.
   - Подсветить P0/P1/P2 правки.

5. **Фактчекинг** — блокирующий gate. (`/blog factcheck <файл>`)
   - Извлечь все числовые утверждения, нормы, штрафы.
   - Сверить с источниками, получить confidence score.
   - Исправить или поставить [нужна проверка] для claims с score < 0.7.
   - После успешного прохождения создать маркер:
     ```
     mkdir -p .claude/factchecked && touch .claude/factchecked/<slug>
     ```
   - **Без маркера pre-commit хук заблокирует коммит** статьи с `draft: false`.

6. **AI-цитируемость** (`/blog geo <файл>`).
   - Оценить passage-level citability под ChatGPT / Perplexity / AI Overviews.
   - Сгенерировать citation capsules (40–60 слов) для каждого H2.
   - Вставить TL;DR-блок, если отсутствует.

7. **Четыре версии контента** (агент `social-media-manager`, обязательно).
   - На основе готовой SEO-статьи подготовить четыре отдельных текста:
     - **Сайт** — уже готов (файл из шага 3).
     - **Дзен** — уникальная статья 3000–5000 знаков с другим углом подачи.
       Не пересказ и не копипаст: иная структура, расширенные примеры,
       практические кейсы. Прямую ссылку на сайт вставить как «источник».
     - **Telegram** — пост 700–900 знаков: крючок, 4 тезиса, ссылка на сайт.
     - **VK** — мини-разбор 1500–3000 знаков, прикрепить hero-image,
       одна ссылка на статью (VK режет охват при > 1 ссылки).
   - Сохранить все версии в `.claude/social/YYYY-MM-DD-slug.md`
     с разделителями `## Сайт`, `## Дзен`, `## Telegram`, `## VK`.

8. **Финализация** (вручную).
   - Перепроверить факты по первоисточникам.
   - Снять `draft: true` → `draft: false`.
   - Убедиться, что маркер `.claude/factchecked/<slug>` создан.
   - Закоммитить в feature-ветку, запушить, открыть PR.

## Категории

Каждая статья помечается одной категорией из `src/consts.ts`:
- `ts-piot`
- `markirovka`
- `zakonodatelstvo`

## Naming convention

- Файл: `YYYY-MM-DD-kebab-slug.md`.
- Slug: транслит ключевого запроса, ≤ 60 символов, без дат.

## Quality gates

- 1500+ слов.
- Все цифры со ссылкой на НПА.
- 3+ внутренних ссылок.
- Frontmatter полный.
- Никаких артефактов английского текста или эмодзи.
- `/blog cannibalization` — нет дублей с > 60% пересечением ключей.
- `/blog factcheck` — нет claims с confidence < 0.7 без пометки + маркер создан.
- `/blog geo` — AI Citation score ≥ 60.
