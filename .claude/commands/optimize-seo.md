# /optimize-seo — SEO-аудит

Запускает агент `seo-optimizer` для проверки оптимизации под Яндекс и Google.

## Цели

- `post` — отдельная статья. Указать slug или путь.
- `category` — все статьи в одной категории (`ts-piot`, `markirovka`,
  `zakonodatelstvo`).
- `site` — общий аудит сайта (мета-теги, перелинковка, разметка).

## Глубина

- `quick` (~15 мин) — title, description, H1/H2, базовая перелинковка.
- `standard` (~45 мин) — полный чек-лист + предложения по разметке.
- `deep` (~90 мин) — анализ кластера, gap-анализ, технический SEO (CWV).

## Что проверяется

1. **Мета.** Title (≤65), description (140–160), keywords, canonical.
2. **Заголовки.** H1 единственный, иерархия H2/H3 логична, ключи в заголовках.
3. **Контент.** Целевой запрос в первом абзаце, плотность 1–2%, LSI.
4. **Разметка.** `Article`, `BreadcrumbList`, при наличии — `FAQPage`, `HowTo`.
5. **Перелинковка.** 3+ внутренних ссылок на родственные материалы.
6. **URL.** Латиница, kebab-case, читаемый slug ≤ 60 символов.
7. **Изображения.** Осмысленный `alt`, формат и размер.
8. **CWV.** LCP < 2.5 c, INP < 200 мс, CLS < 0.1.

## Особенности под Яндекс

- Поведенческие сигналы важнее, чем у Google. Первый экран должен быстро
  закрывать запрос.
- Word-перфект — Яндекс ценит точные совпадения по словоформам ключа.
- Свежесть — `updatedDate` в frontmatter и факт обновления видны Я.Вебмастеру.

## Формат отчёта

```markdown
# SEO-аудит: <URL/slug>

## Что в порядке
- ...

## Что исправить
- P0: ...
- P1: ...
- P2: ...

## Дополнительная разметка
- FAQPage: ...

## Идеи смежных статей для кластера
- ...
```
